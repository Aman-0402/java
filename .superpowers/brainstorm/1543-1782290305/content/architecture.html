Ebook Architecture & Folder Structure

Multi-file, clean separation. index.html = shell. Chapter files = content only. JS loads chapters dynamically.

📁 java-ebook/
├── index.html
Shell: sidebar + header + search + theme

├── 📁 assets/
├── 📁 css/

├── dark.css ← dark mode overrides
└── prism.css ← code highlighting

├── app.js ← nav, chapter loader
├── search.js ← search logic
├── theme.js ← dark mode toggle
└── prism.js ← syntax highlight
└── 📁 img/ ← logo, diagrams

⚙️ How Chapter Loading Works
1. index.html = permanent shell (sidebar + header)
2. Click chapter in sidebar → JS fetches chapters/section1/ch01.html
3. Content injected into #content-area div
4. URL updates: index.html#ch01 (shareable link)
5. Prism.js re-runs → code blocks get highlighted
📄 Chapter File = Content Only

No <html>/<head>/<body> boilerplate. Pure content.
🔍 Search Strategy
data/search-index.json = pre-built keyword map
Each chapter → array of keywords + heading anchors
Search.js filters index on keypress → shows results
Click result → loads chapter + scrolls to heading
Run locally: npx serve . or VS Code Live Server extension — no build step needed.
