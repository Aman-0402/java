📗 Section 1 — Java Basics

Chapter 8: If-Else and Switch Statements

Give your program the ability to make decisions — run different code depending on whether a condition is true or false.

Why Decisions Matter

Imagine a vending machine. When you press a button it checks: "Does this item exist? Is the price correct? Is there stock?" Different answers lead to different actions. Programs work the same way — they constantly check conditions and branch into different paths. Java provides if, else, and switch for exactly this.

The if Statement

An if statement runs a block of code only when a condition is true. If the condition is false, the block is skipped entirely.

public class IfDemo {
    public static void main(String[] args) {
        int temperature = 35;

        if (temperature > 30) {
            System.out.println("It's hot outside — drink water!");
        }

        System.out.println("Program continues here regardless.");
    }
}

The if-else Statement

Add an else block to handle the case when the condition is false. Exactly one of the two blocks will always run.

public class IfElseDemo {
    public static void main(String[] args) {
        int age = 16;

        if (age >= 18) {
            System.out.println("You can vote.");
        } else {
            System.out.println("You are too young to vote.");
        }
    }
}

The if-else if Chain

When you have more than two possible paths, chain multiple conditions using else if. Java checks them in order and runs the first one that is true. The final else is a catch-all that runs if none of the conditions matched.

public class GradeCalculator {
    public static void main(String[] args) {
        int score = 74;
        String grade;

        if (score >= 90) {
            grade = "A";
        } else if (score >= 80) {
            grade = "B";
        } else if (score >= 70) {
            grade = "C";
        } else if (score >= 60) {
            grade = "D";
        } else {
            grade = "F";
        }

        System.out.println("Score: " + score + " -> Grade: " + grade);
        // Output: Score: 74 -> Grade: C
    }
}
💡 Conditions are checked top to bottom. Once one matches, Java skips the rest. So you don't need to write score >= 70 && score < 80 — just score >= 70 is enough because if score were 80 or above, a previous condition would have already matched.

Nested if Statements

You can place an if inside another if. This is called nesting. Use it when one decision depends on the result of another.

public class NestingDemo {
    public static void main(String[] args) {
        boolean hasTicket = true;
        int age = 20;

        if (hasTicket) {
            if (age >= 18) {
                System.out.println("Welcome to the event!");
            } else {
                System.out.println("Sorry, this event is 18+.");
            }
        } else {
            System.out.println("You need a ticket to enter.");
        }
    }
}
⚠️ Don't nest more than 2–3 levels deep. Deeply nested code becomes hard to read. If you find yourself nesting 4+ levels, look for ways to simplify using methods or combined conditions with &&.

The Switch Statement

A switch statement is like a list of labelled doors. You give it a value, and it opens the matching door. It works well when you have one variable with many possible exact values (like a day of the week or a menu choice).

public class DayOfWeek {
    public static void main(String[] args) {
        int day = 3; // 1=Monday, 2=Tuesday, ...

        switch (day) {
            case 1:
                System.out.println("Monday — start of the week");
                break;
            case 2:
                System.out.println("Tuesday");
                break;
            case 3:
                System.out.println("Wednesday — middle of the week");
                break;
            case 4:
                System.out.println("Thursday");
                break;
            case 5:
                System.out.println("Friday — almost the weekend!");
                break;
            case 6:
            case 7:
                System.out.println("Weekend!");  // cases 6 and 7 share this output
                break;
            default:
                System.out.println("Invalid day number");
        }
    }
}
⚠️ Do not forget break at the end of each case! Without it, Java will "fall through" and execute the next case's code too, which is almost never what you want. This is one of the most famous bugs in Java history.
🔍 Fun Fact: The most famous real-world bug caused by a missing break in a switch: Apple's "goto fail" SSL security bug in 2014. A missing statement in Apple's iOS code caused HTTPS security checks to be bypassed silently, leaving millions of iPhones vulnerable to network attacks for months before it was discovered.

Switch Expression (Java 14+)

Modern Java introduced a cleaner switch syntax using -> (arrow). It does not fall through, so you don't need break. It can also return a value directly.

public class ModernSwitch {
    public static void main(String[] args) {
        int day = 6;

        String type = switch (day) {
            case 1, 2, 3, 4, 5 -> "Weekday";
            case 6, 7           -> "Weekend";
            default             -> "Unknown";
        };

        System.out.println("Day " + day + " is a: " + type);
        // Output: Day 6 is a: Weekend
    }
}
💡 The switch expression (with ->) is cleaner and less error-prone than the traditional switch statement (with break). If you are using Java 14 or newer, prefer the arrow syntax.
📜 History: The switch statement has been in Java since version 1.0, but its clunky fall-through syntax caused bugs for decades. Java 14 (2020) introduced the modern switch expression with -> arrows. Java 21 added full pattern matching in switch — a feature that took 25 years to arrive from the original 1996 design.
🌍 Real World: Netflix uses complex conditional logic (if/else chains and switches) in its recommendation engine to decide what content to show you. Every "Because you watched X" recommendation is the result of hundreds of nested conditions evaluating your watch history, time of day, device type, and viewing patterns — running in real-time.

The = vs == Trap in Conditions

This cannot be repeated enough. In a condition inside if ( ... ), you almost always want == to compare, not = to assign.

int x = 5;

// WRONG - assigns 10 to x, condition is always true
if (x = 10) { ... }     // compiler error in Java (unlike C/C++)

// CORRECT - compares x to 10
if (x == 10) { ... }    // returns true or false
ℹ️ Java's compiler actually protects you here — using a single = inside an if condition causes a compile error (because an assignment doesn't return a boolean). But the habit of telling them apart matters greatly in other languages and in other contexts.
