📗 Section 1 — Java Basics

Chapter 19: Collections

The Java Collections Framework gives you ready-made, flexible data structures — dynamic lists, key-value stores, and unique-value sets — so you rarely need to manage raw arrays.

Why Not Just Use Arrays?

Arrays are fast and simple, but they have a critical limitation: their size is fixed when you create them. If you declare int[] scores = new int[5]; and then a sixth student joins, you're stuck. You'd have to create a bigger array and copy everything over manually.

The Java Collections Framework solves this. It provides classes that grow and shrink automatically, search efficiently, and come with dozens of built-in utility methods.

Generics — Telling a Collection What Type to Hold

Collections use generics, written as <TypeName>, to specify what type of data they store. This gives you compile-time type safety — you can't accidentally add an integer to a list of strings. 

ArrayList<String> names = new ArrayList<>();  // holds only Strings
ArrayList<Integer> scores = new ArrayList<>(); // holds only Integers

ArrayList — A Dynamic Array

ArrayList is the most commonly used collection. Think of it as a numbered list that can grow and shrink — like a shopping list where you can always add more items.

🔍 Fun Fact: ArrayList is arguably the most used Java class in codebases worldwide. Under the hood, it is just an array that doubles in size when full. When the internal array fills up, ArrayList creates a new array twice as large and copies all elements — this is O(n) but happens rarely enough that the amortised cost is O(1) per add.
import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        ArrayList<String> fruits = new ArrayList<>();

        // Adding elements
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Cherry");
        fruits.add("Apple");   // duplicates allowed

        System.out.println("Size: " + fruits.size());       // 4
        System.out.println("First: " + fruits.get(0));      // Apple
        System.out.println("Contains Banana: " + fruits.contains("Banana")); // true

        // Remove by value
        fruits.remove("Banana");

        // Iterate with for-each
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}

🔍 How This Code Works

  • ArrayList<String> fruits = new ArrayList<>(); — the diamond <> on the right lets Java infer the type from the left side; the list starts empty and will grow automatically
  • fruits.add("Apple"); — appends an element to the end; the list resizes its internal array automatically if needed
  • fruits.add("Apple"); (second time) — ArrayList allows duplicate values; both "Apple" entries are stored
  • fruits.size() — returns the current number of elements; unlike arrays, this reflects the live count, not a fixed capacity
  • fruits.get(0) — retrieves the element at index 0 (zero-based), just like an array
  • fruits.remove("Banana"); — removes the first occurrence of the matching object; elements shift left to fill the gap
  • for (String fruit : fruits) — the enhanced for-each loop iterates over all remaining elements in insertion order

Output:

Size: 4
First: Apple
Contains Banana: true
Apple
Cherry
Apple

HashMap — Key-Value Pairs

A HashMap is like a dictionary: every entry has a key (the word) and a value (the definition). Keys must be unique — looking up a key gives you its value instantly, no matter how many entries exist. 

import java.util.HashMap;
import java.util.Map;

public class Main {
    public static void main(String[] args) {
        HashMap<String, Integer> scores = new HashMap<>();

        scores.put("Alice", 95);
        scores.put("Bob",   87);
        scores.put("Carol", 91);

        // Get a value by key
        System.out.println("Alice's score: " + scores.get("Alice"));

        // Check if key exists
        System.out.println("Has Dave? " + scores.containsKey("Dave"));

        // Iterate over all entries
        for (Map.Entry<String, Integer> entry : scores.entrySet()) {
            System.out.println(entry.getKey() + " → " + entry.getValue());
        }
    }
}

🔍 How This Code Works

  • HashMap<String, Integer> scores = new HashMap<>(); — the first type parameter is the key type, the second is the value type; keys must be unique
  • scores.put("Alice", 95); — inserts a key-value pair; if the key already exists, the old value is replaced
  • scores.get("Alice") — looks up the value by key in O(1) average time using a hash function; returns null if the key is not found
  • scores.containsKey("Dave") — checks for key existence without retrieving the value; returns a boolean
  • scores.entrySet() — returns a Set of Map.Entry objects, each holding one key-value pair
  • entry.getKey() and entry.getValue() — extract the key and value from each entry during iteration

Output:

Alice's score: 95
Has Dave? false
Alice → 95
Bob → 87
Carol → 91
💡 Tip: Use entrySet() to loop through a HashMap when you need both the key and the value. Use keySet() when you only need keys, and values() when you only need values.
🌍 Real World: Twitter's trending topics are computed using a structure like HashMap<String, Integer> — mapping each hashtag to its count. When you tweet #JavaIsAwesome, a counter in a HashMap increments. Every few seconds, the top entries are ranked and pushed to the trending list you see on screen.

HashSet — Unique Values Only

A HashSet is like a bag that automatically rejects duplicates. If you try to add the same item twice, the second add is silently ignored. Order is not guaranteed. 

import java.util.HashSet;

public class Main {
    public static void main(String[] args) {
        HashSet<String> tags = new HashSet<>();

        tags.add("java");
        tags.add("programming");
        tags.add("java");    // duplicate — ignored
        tags.add("oop");

        System.out.println("Unique tags: " + tags.size());  // 3
        System.out.println(tags);   // order may vary
        System.out.println("Has 'java'? " + tags.contains("java")); // true
    }
}

🔍 How This Code Works

  • HashSet<String> tags = new HashSet<>(); — a set that uses hashing internally; guarantees uniqueness of elements
  • tags.add("java"); tags.add("java"); — the second add is silently ignored; the set already contains "java", so its size stays the same
  • tags.size() — returns 3, not 4, because the duplicate was rejected
  • System.out.println(tags); — prints the set's contents; the order may differ on each run because HashSet does not preserve insertion order
  • tags.contains("java") — membership check in O(1) average time; far faster than searching an ArrayList for large collections

LinkedList — Brief Overview

A LinkedList also implements the List interface (like ArrayList) but stores elements as chained nodes. It is faster for frequent insertions and deletions at the beginning or middle, but slower for random access by index. For most beginner use cases, ArrayList is the better default.

When to Use Which Collection

  • ArrayList — ordered list, duplicates OK, fast access by index. Default list choice.
  • HashMap — fast lookup by a unique key. Use for counts, mappings, caches.
  • HashSet — need unique values only, order doesn't matter. Use for membership testing.
  • LinkedList — frequent inserts/deletes at list ends. Rare in typical code.
⚠️ Common mistake: Using int instead of Integer as a generic type. Generics require wrapper types (capitalized): Integer, Double, Boolean — not the primitives int, double, boolean.
ℹ️ Always import from java.util.* (or the specific class like java.util.ArrayList). The Collections Framework lives in that package. Your IDE will suggest the import automatically.
📜 History: Java's Collections Framework was redesigned in Java 2 (1998) by Joshua Bloch. He later wrote Effective Java — the most influential Java book ever written. The framework he designed (List, Set, Map interfaces with ArrayList, HashMap, HashSet implementations) has remained largely unchanged for over 25 years.

🧠 Multiple Choice Questions

1. Which collection does NOT allow duplicate values?

  • A) ArrayList
  • B) LinkedList
  • C) HashSet
  • D) ArrayDeque
Show Answer

✅ C) HashSet — Sets reject duplicate elements. set.add("apple"); set.add("apple"); results in only one "apple".

2. What does HashMap.get("key") return if the key doesn't exist?

  • A) 0
  • B) ""
  • C) null
  • D) throws KeyNotFoundException
Show Answer

✅ C) null — HashMap.get() returns null for missing keys. Use getOrDefault("key", defaultValue) to avoid null checks.

3. Which interface do ArrayList and LinkedList both implement?

  • A) Map
  • B) Set
  • C) List
  • D) Collection only
Show Answer

✅ C) List — Both implement java.util.List, which extends Collection. This is why you can write List<String> list = new ArrayList<>();.

4. What is the time complexity of HashMap.get() in the average case?

  • A) O(n)
  • B) O(log n)
  • C) O(1)
  • D) O(n²)
Show Answer

✅ C) O(1) — HashMap uses hashing for constant-time lookups in the average case. Worst case (many collisions) is O(n) or O(log n) in Java 8+.
