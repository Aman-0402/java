📗 Section 1 — Java Basics

Chapter 19: Collections

The Java Collections Framework gives you ready-made, flexible data structures — dynamic lists, key-value stores, and unique-value sets — so you rarely need to manage raw arrays.

Why Not Just Use Arrays?

Arrays are fast and simple, but they have a critical limitation: their size is fixed when you create them. If you declare int[] scores = new int[5]; and then a sixth student joins, you're stuck. You'd have to create a bigger array and copy everything over manually.

The Java Collections Framework solves this. It provides classes that grow and shrink automatically, search efficiently, and come with dozens of built-in utility methods.

Generics — Telling a Collection What Type to Hold

Collections use generics, written as <TypeName>, to specify what type of data they store. This gives you compile-time type safety — you can't accidentally add an integer to a list of strings. 

ArrayList<String> names = new ArrayList<>();  // holds only Strings
ArrayList<Integer> scores = new ArrayList<>(); // holds only Integers

ArrayList — A Dynamic Array

ArrayList is the most commonly used collection. Think of it as a numbered list that can grow and shrink — like a shopping list where you can always add more items. 

import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        ArrayList<String> fruits = new ArrayList<>();

        // Adding elements
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Cherry");
        fruits.add("Apple");   // duplicates allowed

        System.out.println("Size: " + fruits.size());       // 4
        System.out.println("First: " + fruits.get(0));      // Apple
        System.out.println("Contains Banana: " + fruits.contains("Banana")); // true

        // Remove by value
        fruits.remove("Banana");

        // Iterate with for-each
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}

Output:

Size: 4
First: Apple
Contains Banana: true
Apple
Cherry
Apple

HashMap — Key-Value Pairs

A HashMap is like a dictionary: every entry has a key (the word) and a value (the definition). Keys must be unique — looking up a key gives you its value instantly, no matter how many entries exist. 

import java.util.HashMap;
import java.util.Map;

public class Main {
    public static void main(String[] args) {
        HashMap<String, Integer> scores = new HashMap<>();

        scores.put("Alice", 95);
        scores.put("Bob",   87);
        scores.put("Carol", 91);

        // Get a value by key
        System.out.println("Alice's score: " + scores.get("Alice"));

        // Check if key exists
        System.out.println("Has Dave? " + scores.containsKey("Dave"));

        // Iterate over all entries
        for (Map.Entry<String, Integer> entry : scores.entrySet()) {
            System.out.println(entry.getKey() + " → " + entry.getValue());
        }
    }
}

Output:

Alice's score: 95
Has Dave? false
Alice → 95
Bob → 87
Carol → 91
💡 Tip: Use entrySet() to loop through a HashMap when you need both the key and the value. Use keySet() when you only need keys, and values() when you only need values.

HashSet — Unique Values Only

A HashSet is like a bag that automatically rejects duplicates. If you try to add the same item twice, the second add is silently ignored. Order is not guaranteed. 

import java.util.HashSet;

public class Main {
    public static void main(String[] args) {
        HashSet<String> tags = new HashSet<>();

        tags.add("java");
        tags.add("programming");
        tags.add("java");    // duplicate — ignored
        tags.add("oop");

        System.out.println("Unique tags: " + tags.size());  // 3
        System.out.println(tags);   // order may vary
        System.out.println("Has 'java'? " + tags.contains("java")); // true
    }
}

LinkedList — Brief Overview

A LinkedList also implements the List interface (like ArrayList) but stores elements as chained nodes. It is faster for frequent insertions and deletions at the beginning or middle, but slower for random access by index. For most beginner use cases, ArrayList is the better default.

When to Use Which Collection

  • ArrayList — ordered list, duplicates OK, fast access by index. Default list choice.
  • HashMap — fast lookup by a unique key. Use for counts, mappings, caches.
  • HashSet — need unique values only, order doesn't matter. Use for membership testing.
  • LinkedList — frequent inserts/deletes at list ends. Rare in typical code.
⚠️ Common mistake: Using int instead of Integer as a generic type. Generics require wrapper types (capitalized): Integer, Double, Boolean — not the primitives int, double, boolean.
ℹ️ Always import from java.util.* (or the specific class like java.util.ArrayList). The Collections Framework lives in that package. Your IDE will suggest the import automatically.
