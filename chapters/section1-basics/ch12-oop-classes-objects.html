📗 Section 1 — Java Basics

Chapter 12: OOP — Classes and Objects

Object-Oriented Programming lets you model real-world things as "objects" built from "blueprints" called classes.

What Is OOP and Why Does It Exist?

Imagine you're building a housing estate. Before the first brick is laid, an architect draws a blueprint. The blueprint describes everything a house will have — number of rooms, door positions, window sizes. From that single blueprint you can build hundreds of identical houses, each with their own address, colour, and furniture.

In Java, a class is the blueprint, and an object is one house built from that blueprint. You define a class once, then create as many objects (instances) as you like. Each object has its own copy of the data described in the class.

Declaring a Class

A class is declared with the class keyword. Inside the class you list the data it holds (called fields or instance variables) and the actions it can perform (called methods, covered in later chapters). 

// A blueprint that describes every Car
public class Car {
    // Fields — every Car object will have these three pieces of data
    String make;
    String model;
    int    year;
}
💡 Tip: By convention, class names start with an uppercase letter (Car, not car). Each word is capitalised — this style is called PascalCase.
📜 History: Object-Oriented Programming was invented by Ole-Johan Dahl and Kristen Nygaard in 1967 with the Simula language. They needed to simulate ships and people in a harbour — so they created "objects" to model real-world entities. Java, born 28 years later, is their spiritual successor.

Creating Objects with new

The new keyword tells Java to build a fresh object in memory using the class as its blueprint. The result is stored in a reference variable whose type is the class name. 

public class Main {
    public static void main(String[] args) {
        // Create two Car objects from the same blueprint
        Car car1 = new Car();
        Car car2 = new Car();

        // Give each object its own data using dot notation
        car1.make  = "Toyota";
        car1.model = "Corolla";
        car1.year  = 2022;

        car2.make  = "Honda";
        car2.model = "Civic";
        car2.year  = 2023;

        // Read the data back
        System.out.println(car1.make + " " + car1.model + " (" + car1.year + ")");
        System.out.println(car2.make + " " + car2.model + " (" + car2.year + ")");
    }
}

Output: 

Toyota Corolla (2022)
Honda Civic (2023)

Dot Notation

You reach inside an object using a dot: objectName.fieldName. Think of it like an address — "the make that belongs to car1".

⚠️ Common mistake: Forgetting new and writing Car car1 = Car(); causes a compile error. The new keyword is what actually allocates memory and creates the object.
ℹ️ How many objects can you create? As many as your computer's memory allows. A single class definition can be the blueprint for thousands of objects running in the same program.
🔍 Fun Fact: Everything in Java (except primitives) is an object. When you write String name = "Java", name is an object with methods like .length() and .toUpperCase(). Even null is a special reference — pointing to no object at all.

Full Runnable Example

// File: Car.java
public class Car {
    String make;
    String model;
    int    year;

    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.make  = "Ford";
        myCar.model = "Mustang";
        myCar.year  = 1969;

        System.out.println("Make  : " + myCar.make);
        System.out.println("Model : " + myCar.model);
        System.out.println("Year  : " + myCar.year);
    }
}
🌍 Real World: The game Minecraft is written in Java. Every block, every mob (creeper, zombie, skeleton), every item is a Java class. There are over 300 entity classes in Minecraft's codebase — and OOP is what makes managing all that complexity possible.
