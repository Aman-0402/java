📗 Section 1 — Java Basics

Chapter 10: Arrays

Store and manage multiple values of the same type in a single, ordered container — and access each one by its position number.

What is an Array?

Imagine a row of labelled post boxes in an apartment building: Box 0, Box 1, Box 2, Box 3... Each box holds one item, and you reach any box by its number. An array in Java works exactly like this — it is a fixed-length row of slots, each holding a value of the same type, each accessible by its position (called an index).

Without arrays, storing the scores of 30 students would need 30 separate variables. With an array, you need just one.

Declaring and Initialising an Array

There are two common ways to create an array:

public class ArrayCreation {
    public static void main(String[] args) {
        // Method 1: declare size, fill values later
        int[] scores = new int[5];  // 5 slots, all set to 0
        scores[0] = 88;
        scores[1] = 72;
        scores[2] = 95;
        scores[3] = 61;
        scores[4] = 79;

        // Method 2: declare and initialise at the same time (array literal)
        String[] days = {"Monday", "Tuesday", "Wednesday", "Thursday", "Friday"};

        System.out.println(scores[2]);  // 95
        System.out.println(days[0]);    // Monday
    }
}
⚠️ Arrays in Java are zero-indexed. The first element is at index 0, not 1. An array of size 5 has valid indices 0, 1, 2, 3, 4. Trying to access index 5 causes an ArrayIndexOutOfBoundsException.
🔍 Fun Fact: Arrays start at index 0 because in C (which influenced Java), the index represents a memory offset from the start of the array. array[0] literally means "0 bytes from the start." Dijkstra wrote a famous 1982 paper arguing why 0-based indexing is mathematically superior to 1-based — and he won the debate for most languages.

The length Property

array.length tells you how many slots the array has. It is a property, not a method — notice there are no parentheses after it.

public class ArrayLength {
    public static void main(String[] args) {
        double[] prices = {1.99, 4.50, 12.75, 0.99, 7.00};

        System.out.println("Number of prices: " + prices.length);  // 5

        // Access the last element using length - 1
        System.out.println("Last price: " + prices[prices.length - 1]);  // 7.0
    }
}
💡 To get the last element of any array, use array[array.length - 1]. This works regardless of the array size.

Iterating an Array with a for Loop

Combining arrays with loops is where they become truly powerful. You can process every element without writing repetitive code.

public class LoopThroughArray {
    public static void main(String[] args) {
        int[] scores = {88, 72, 95, 61, 79};
        int total = 0;

        for (int i = 0; i < scores.length; i++) {
            total += scores[i];
            System.out.println("Score at index " + i + ": " + scores[i]);
        }

        double average = (double) total / scores.length;
        System.out.printf("Average score: %.2f%n", average);
    }
}

The For-Each Loop with Arrays

When you just need to read through every element and don't need the index, the enhanced for loop (for-each) is cleaner:

public class ForEachArray {
    public static void main(String[] args) {
        String[] fruits = {"Apple", "Banana", "Cherry", "Date"};

        for (String fruit : fruits) {  // "for each fruit in fruits"
            System.out.println("Fruit: " + fruit);
        }
    }
}
ℹ️ Use the regular for loop when you need the index (e.g., to modify elements or compare adjacent elements). Use for-each when you only need to read each value — it is shorter and harder to get wrong.

Common Error: ArrayIndexOutOfBoundsException

This is Java's way of telling you that you tried to access an index that doesn't exist in the array.

public class BoundsError {
    public static void main(String[] args) {
        int[] data = {10, 20, 30};  // valid indices: 0, 1, 2

        System.out.println(data[0]);  // 10 - OK
        System.out.println(data[2]);  // 30 - OK
        System.out.println(data[3]);  // ERROR! Index 3 doesn't exist
        // Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: Index 3 out of bounds for length 3
    }
}
⚠️ When you see ArrayIndexOutOfBoundsException, immediately check two things: the size of your array, and the index you used. The most common cause is writing i <= array.length in a loop instead of i < array.length (off-by-one error).
📜 History: The ArrayIndexOutOfBoundsException is one of the most famous Java errors — and it was intentional. In 1990s C/C++, reading past the end of an array caused silent memory corruption and security vulnerabilities (entire categories of hacks exploited this). Java's bounds checking was a deliberate safety feature, at the cost of a small performance penalty.

2D Arrays (Intro)

A 2D array is like a table with rows and columns — think of a spreadsheet or a chess board. You declare it with two pairs of square brackets.

public class TwoDArray {
    public static void main(String[] args) {
        // 3 rows, 3 columns
        int[][] grid = {
            {1, 2, 3},
            {4, 5, 6},
            {7, 8, 9}
        };

        // Access element at row 1, column 2
        System.out.println(grid[1][2]);  // 6

        // Print the entire grid with nested loops
        for (int row = 0; row < grid.length; row++) {
            for (int col = 0; col < grid[row].length; col++) {
                System.out.printf("%3d", grid[row][col]);
            }
            System.out.println();
        }
        // Output:
        //   1  2  3
        //   4  5  6
        //   7  8  9
    }
}
💡 For 2D arrays: grid.length gives the number of rows. grid[0].length gives the number of columns in the first row. Use nested for loops to traverse all elements.
🌍 Real World: Every image on your screen is stored as a 2D array of pixels. A 1920×1080 HD display is essentially an int[1080][1920] array where each int stores the RGB color value of one pixel. Java's image processing libraries (like BufferedImage) use exactly this structure to manipulate photos and graphics.
