📗 Section 1 — Java Basics

Chapter 4: Hello, World! — Anatomy of a Java Program

Dissect every single word of the classic Hello World program so you know exactly why each piece is there and what happens when you run it.

The Full Program

Here is the program you will write. Do not worry if it looks like a wall of words right now — by the end of this chapter, every word will make sense.

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

🔍 How This Code Works

  • public class HelloWorld — defines a class (a blueprint/container); every Java file must have exactly one public class matching the filename
  • public static void main(String[] args) — the exact method signature the JVM searches for as the program's entry point; all five words are required
  • static — lets the JVM call main without first creating a HelloWorld object; without it the program won't start
  • void — declares that main returns no value; the JVM does not expect one back
  • String[] args — an array of strings the user can pass on the command line (e.g. java HelloWorld foo bar); beginners rarely use this but it must still be present
  • System.out.println("Hello, World!") — sends the text to standard output; the semicolon at the end is mandatory on every Java statement
  • The outer { } wraps the class body; the inner { } wraps the method body — every opening brace must have a matching closing brace

What is a Class?

Imagine a class as a blueprint. A house blueprint is not a house — it describes how a house should be built. Similarly, a Java class is a blueprint that describes a piece of your program. Every piece of Java code must live inside a class. Right now, your entire program is one class called HelloWorld.

The keyword public before class means "anyone can see and use this class." You will learn more about public vs private in the Object-Oriented chapters. For now, just always write public class.

⚠️ The class name must exactly match the filename. If your class is named HelloWorld, the file must be saved as HelloWorld.java — same capitalisation, no spaces. Java is case-sensitive: helloworld and HelloWorld are completely different names.
📜 History: The tradition of printing "Hello, World!" as the first program started with Brian Kernighan's 1972 book "A Tutorial Introduction to the Language B." It became iconic in C and then spread to virtually every programming language ever created. When you type it in Java, you are part of a 50-year programmer tradition.

Breaking Down public static void main(String[] args)

This line is called the main method signature. It is the entry point — the starting line of your program. When you run a Java program, the Java runtime looks for exactly this line and begins executing from there. Let's decode each word:

WordWhat it means
publicAny code, anywhere, can call this method — including the Java runtime itself.
staticThis method belongs to the class itself, not to an object created from the class. The runtime can call it without creating a HelloWorld object first.
voidThis method does not return any value when it finishes. (Some methods return a number or text; this one just runs and stops.)
mainThe name the Java runtime specifically looks for as the entry point. It must be spelled exactly main in lowercase.
String[] argsA list (array) of text values you can pass in when you run the program from the terminal. Beginners rarely use this, but it must still be there.
ℹ️ You do not need to memorise why every word is there right now. Just accept the whole line as a magic phrase you type every time you start a new Java program. The understanding will build naturally over the next few chapters.
🔍 Fun Fact: Every Java program needs exactly one public static void main(String[] args) as its entry point. This design was intentional — it makes every Java program self-contained and runnable without any external configuration. Compare that to JavaScript, where there is no standard entry point at all.

What Does System.out.println() Do?

Think of it as a walkie-talkie that sends messages to your screen. Breaking it down:

  • System — a built-in Java class that represents your computer's system resources.
  • out — the "output channel" of that system (the screen/terminal).
  • println — short for "print line." It prints whatever you put inside the parentheses, then moves to a new line.

The difference between println and print:

public class PrintDemo {
    public static void main(String[] args) {
        System.out.println("Line 1");  // prints then moves to next line
        System.out.println("Line 2");  // prints on a new line
        System.out.print("No ");       // prints, but stays on same line
        System.out.print("newline!");  // continues on same line
    }
}

🔍 How This Code Works

  • System.out.println("Line 1") — prints the text and appends a newline character \n, so the next output starts on a fresh line
  • System.out.print("No ") — prints the text without any newline; the cursor stays at the end of the same line
  • The trailing space inside "No " is intentional — without it "No" and "newline!" would print with no gap between them
  • // prints then moves to next line — double-slash starts a single-line comment; Java ignores everything after // to the end of the line
  • Both print and println belong to System.out, which is a PrintStream connected to the terminal

Output:

Line 1
Line 2
No newline!

Compiling and Running — What Actually Happens

Java uses a two-step process before your code runs:

  1. Compile — javac HelloWorld.java reads your human-readable code and translates it into bytecode (a file called HelloWorld.class). Bytecode is not machine code — it is a special format designed to run on the Java Virtual Machine (JVM).
  2. Run — java HelloWorld starts the JVM, which reads the .class file and actually executes your program.
// Compile in terminal:  javac HelloWorld.java  -> creates HelloWorld.class
// Run in terminal:      java HelloWorld         -> JVM executes HelloWorld.class

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

🔍 How This Code Works

  • javac HelloWorld.java — the compiler command; javac stands for "Java compiler"; it produces HelloWorld.class
  • HelloWorld.class — the resulting bytecode file; this is platform-neutral and runs on any operating system with a JVM installed
  • java HelloWorld — the runtime command; starts the JVM and loads HelloWorld.class (omit the .class extension here)
  • The JVM reads bytecode and translates it to machine code for your specific OS at runtime — this is "Write Once, Run Anywhere"
  • Comments starting with // explain the shell commands; they are inside a Java code block here for display purposes only
💡 This two-step process is why Java programs run on any operating system without change. The .class bytecode is the same on Windows, macOS, and Linux. The JVM on each platform handles the differences — that is what "Write Once, Run Anywhere" means.
🌍 Real World: The System.out.println() method you write in Hello World is the exact same method used in production at Amazon, Twitter, and LinkedIn's backend systems — just printing to server logs instead of your terminal. Your first program uses real, production-grade Java.

Common Errors and What They Mean

Error 1 — Wrong filename

// File is saved as helloworld.java, but class name is HelloWorld
public class HelloWorld { }
// Compiler error: class HelloWorld is public, should be declared in a file named HelloWorld.java

Error 2 — Missing semicolon

System.out.println("Hello")   // no semicolon!
// Compiler error: ';' expected

Every statement in Java must end with a semicolon ; — think of it as a full stop (period) at the end of a sentence.

Error 3 — Mismatched curly braces

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
// Missing closing } for the class block
// Compiler error: reached end of file while parsing
⚠️ Every opening brace { must have a matching closing brace }. Count your braces! VS Code highlights matching brackets when your cursor is next to one — use that to check.
