📗 Section 1 — Java Basics

Chapter 5: Variables and Data Types

Learn how to store information in your program using variables, and understand the different kinds of data Java can hold.

What is a Variable?

Imagine a labelled box. You write a name on the outside of the box (like "age"), and you put a value inside (like 25). At any point, you can open the box to look at the value, or replace it with a different value. That is exactly what a variable is in Java — a named container that holds a piece of data.

Declaring a Variable

To create a variable in Java, you write the data type first, then the name, then optionally give it a starting value:

public class VariableDemo {
    public static void main(String[] args) {
        // Declare and assign in one line
        int age = 25;
        double price = 9.99;
        boolean isStudent = true;
        char grade = 'A';
        String name = "Alice";

        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Price: $" + price);
        System.out.println("Is student: " + isStudent);
        System.out.println("Grade: " + grade);
    }
}

Primitive Data Types

A primitive type stores a simple, raw value directly. Java has eight primitive types, but you will use these four most often as a beginner:

TypeStoresExampleSize
intWhole numbersint score = 100;-2 billion to 2 billion
doubleDecimal numbersdouble pi = 3.14159;15+ decimal digits of precision
booleanTrue or falseboolean done = false;true or false only
charOne characterchar letter = 'Z';Any single Unicode character
💡 Notice that char values use single quotes ('A'), while text strings use double quotes ("Hello"). Mixing them up is a very common mistake.
🔍 Fun Fact: Java has exactly 8 primitive data types — and they have not changed since Java 1.0 in 1996. The designers wanted stability above all. The same int keyword works identically in 1996 Java and 2024 Java. Almost no other language can claim this level of backwards compatibility.
📜 History: Java borrowed its primitive types from C/C++. But unlike C, Java guarantees that int is always exactly 32 bits on every platform. In C, the size of int varies by operating system — which caused serious portability bugs. Locking down the sizes was a deliberate Java design decision to make "Write Once, Run Anywhere" actually work.

The String Type

String is different from the primitives above — it is a reference type, meaning it is actually an object (a complex thing built from many parts). But you can use it almost the same way as a primitive. A String holds any sequence of characters — words, sentences, even numbers stored as text.

public class StringDemo {
    public static void main(String[] args) {
        String firstName = "Bob";
        String lastName = "Smith";

        // Join (concatenate) strings with +
        String fullName = firstName + " " + lastName;
        System.out.println("Full name: " + fullName);

        // String length
        System.out.println("Length: " + fullName.length());

        // Convert to uppercase
        System.out.println("Uppercase: " + fullName.toUpperCase());
    }
}

Declaring vs Assigning vs Reassigning

Declaring means creating the box (telling Java the type and name). Assigning means putting a value in the box. Reassigning means replacing the value with a new one.

public class ReassignDemo {
    public static void main(String[] args) {
        int count;           // Declare (box is empty)
        count = 0;           // Assign (put 0 in the box)
        count = count + 1;   // Reassign (replace with current value + 1)
        System.out.println(count); // prints 1

        count = 10;
        System.out.println(count); // prints 10
    }
}
⚠️ You cannot use a variable before assigning it a value. int x; System.out.println(x); will give a compiler error: "variable x might not have been initialized."
🌍 Real World: A bank's account balance is stored as long or BigDecimal — never double. Why? Because double has floating-point precision errors: 0.1 + 0.2 in Java returns 0.30000000000000004. For money, that rounding error is catastrophic. Financial systems learned this the hard way.

The var Keyword (Java 10+)

Since Java 10, you can use var instead of writing the type explicitly. Java will figure out the type automatically by looking at the value you assign. This is called type inference.

public class VarDemo {
    public static void main(String[] args) {
        var score = 95;          // Java infers: int
        var temperature = 36.6;  // Java infers: double
        var city = "London";     // Java infers: String
        var active = true;       // Java infers: boolean

        System.out.println(city + ": " + temperature + " degrees");
    }
}
ℹ️ var does NOT make Java loosely typed. The type is still fixed at the moment you declare the variable — Java just figures it out for you so you don't have to type it. You cannot change the type later (e.g., store a String in a variable that started as an int).

Naming Conventions

Java variable names follow camelCase: start with a lowercase letter, and capitalise the first letter of every subsequent word. No spaces, no hyphens.

// Good names (camelCase)
int studentAge = 20;
double accountBalance = 1500.75;
boolean isEmailVerified = false;
String userFirstName = "Maria";

// Bad names (avoid these)
int StudentAge = 20;    // starts with uppercase (that's for class names)
double account_balance; // underscores are for constants, not variables
int 2fast;              // cannot start with a number
💡 Choose meaningful names. int age is clear; int a is mysterious. Future-you (and any teammate) will thank you when reading the code weeks later.

🧠 Multiple Choice Questions

1. Which data type should you use to store the value 3.14159?

  • A) int
  • B) char
  • C) boolean
  • D) double
Show Answer

✅ D) double — double stores 64-bit floating-point numbers. Use float for 32-bit (less precision).

2. What is the default value of an uninitialized int field in a Java class?

  • A) null
  • B) 1
  • C) 0
  • D) undefined
Show Answer

✅ C) 0 — Java initializes numeric fields to 0, booleans to false, and objects to null by default.

3. What keyword makes a variable's value unchangeable after assignment?

  • A) static
  • B) const
  • C) final
  • D) immutable
Show Answer

✅ C) final — final int MAX = 100; creates a constant. Trying to reassign it causes a compile error.

4. Which of these is NOT a valid Java variable name?

  • A) myAge
  • B) _count
  • C) 2fast
  • D) firstName
Show Answer

✅ C) 2fast — Variable names cannot start with a digit. They must start with a letter, _, or $.
