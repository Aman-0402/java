📗 Section 1 — Java Basics

Chapter 9: Loops

Stop repeating yourself — let Java do the repetitive work by executing a block of code over and over until a condition is met.

Why Loops Exist

Imagine you need to print "Good morning!" 1000 times. Writing 1000 println statements would be absurd. A loop solves this by running the same block of code repeatedly. The key principle programmers live by: Don't Repeat Yourself (DRY). Loops are the most direct way to honour that principle.

The for Loop

Use a for loop when you know exactly how many times you want to repeat something. It has three parts in the parentheses, separated by semicolons:

  1. Initialise — run once before the loop starts (e.g., int i = 0)
  2. Condition — checked before each iteration; loop stops when false (e.g., i < 5)
  3. Update — runs after each iteration (e.g., i++)
public class ForLoopDemo {
    public static void main(String[] args) {
        // Print 0, 1, 2, 3, 4
        for (int i = 0; i < 5; i++) {
            System.out.println("i = " + i);
        }

        // Count down
        for (int n = 10; n >= 1; n--) {
            System.out.print(n + " ");
        }
        System.out.println("Blast off!");

        // Step by 2
        for (int even = 0; even <= 10; even += 2) {
            System.out.print(even + " ");  // 0 2 4 6 8 10
        }
    }
}

🔍 How This Code Works

  • for (int i = 0; i < 5; i++) — three-part header: initialise (int i = 0), condition (i < 5), update (i++)
  • int i = 0 — the initialiser runs once before the loop starts; i is scoped to the loop and not accessible outside
  • i < 5 — checked before every iteration; when it becomes false (i reaches 5), the loop exits
  • i++ — the update expression runs after each iteration body; increments i by 1
  • n-- in the countdown loop — decrements n by 1 each iteration, counting from 10 down to 1
  • even += 2 — increments by 2 instead of 1, producing only even numbers; the update expression can be any valid statement
💡 The variable i in a for loop is called the loop counter or index. By convention, short names like i, j, k are perfectly fine inside loops — everyone reading Java code knows what they mean.
📜 History: The for loop syntax (for (init; condition; update)) was adopted directly from C (1972). James Gosling kept it intentionally familiar so that C and C++ developers could learn Java quickly. This single decision accelerated Java's adoption massively during the late 1990s developer boom.

The while Loop

A while loop keeps running as long as a condition is true. Use it when you don't know in advance how many times you need to loop — for example, repeating until the user enters valid input.

import java.util.Scanner;

public class WhileDemo {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int number = 0;

        while (number <= 0) {
            System.out.print("Enter a positive number: ");
            number = sc.nextInt();
            if (number <= 0) {
                System.out.println("That's not positive. Try again.");
            }
        }

        System.out.println("You entered: " + number);
        sc.close();
    }
}
⚠️ The infinite loop trap: if the condition never becomes false, the loop runs forever and freezes your program. Always make sure something inside your loop eventually makes the condition false. If your program seems frozen, press Ctrl+C in the terminal to stop it.
🔍 Fun Fact: Infinite loops are not always bugs. Web servers run in an intentional infinite loop — while (true) { acceptConnection(); } — continuously waiting for incoming requests. Every time your browser loads a webpage, it hits a server running that infinite loop.

The do-while Loop

A do-while loop is like a while loop, but it runs the body at least once before checking the condition. This is useful when you always need to execute the block once (like showing a menu to the user).

import java.util.Scanner;

public class DoWhileDemo {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int choice;

        do {
            System.out.println("Menu: 1=Play  2=Settings  3=Quit");
            System.out.print("Your choice: ");
            choice = sc.nextInt();
        } while (choice != 3);  // keep looping until user picks Quit

        System.out.println("Goodbye!");
        sc.close();
    }
}

break and continue

break exits the loop immediately. continue skips the rest of the current iteration and jumps to the next one.

public class BreakContinueDemo {
    public static void main(String[] args) {
        // break: stop as soon as we find 7
        for (int i = 1; i <= 10; i++) {
            if (i == 7) {
                System.out.println("Found 7, stopping.");
                break;  // exits the loop
            }
            System.out.print(i + " ");
        }
        // Output: 1 2 3 4 5 6 Found 7, stopping.

        System.out.println();

        // continue: skip odd numbers, only print evens
        for (int i = 1; i <= 10; i++) {
            if (i % 2 != 0) {
                continue;  // skip this iteration
            }
            System.out.print(i + " ");
        }
        // Output: 2 4 6 8 10
    }
}

🔍 How This Code Works

  • break — immediately exits the innermost loop; execution continues with the statement after the loop's closing brace
  • if (i == 7) { ... break; } — the if check must come before the System.out.print so 7 itself is never printed
  • continue — skips the rest of the current loop body and jumps directly to the update expression (i++), then checks the condition again
  • i % 2 != 0 — true when i is odd (remainder is 1); continue then skips the print, so odd numbers are never printed
  • System.out.println() with no argument — prints an empty line (just a newline), used here to separate the two loop outputs
  • Both break and continue only affect the innermost loop when used inside nested loops

The For-Each Loop (Preview)

Java also has a for-each loop (officially called the enhanced for loop) designed for reading through collections and arrays. You will use it extensively in the next chapter on arrays. Here is a quick preview:

public class ForEachPreview {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30, 40, 50};

        for (int num : numbers) {  // "for each num in numbers"
            System.out.println(num);
        }
    }
}

Nested Loops

You can put a loop inside another loop. The inner loop completes all its iterations for each single iteration of the outer loop. A classic example: printing a multiplication table.

public class NestedLoops {
    public static void main(String[] args) {
        // Print a 3x3 multiplication table
        for (int row = 1; row <= 3; row++) {
            for (int col = 1; col <= 3; col++) {
                System.out.printf("%4d", row * col);
            }
            System.out.println(); // new line after each row
        }
        // Output:
        //    1   2   3
        //    2   4   6
        //    3   6   9
    }
}
ℹ️ For each iteration of the outer loop (row), the inner loop (col) runs 3 times. So the total number of iterations is 3 x 3 = 9. Be careful with nested loops on large numbers — the iteration count multiplies quickly.
🌍 Real World: Google's search crawler (Googlebot) uses loops to visit billions of web pages continuously. It runs 24/7, looping through queues of URLs, downloading pages, and indexing their content. Java loops — just like the ones you are writing — are at the core of how the entire web gets indexed.
⚠️ Common infinite loop patterns to avoid: while (true) { } without a break, or a for loop where the update goes in the wrong direction (e.g., for (int i = 0; i < 10; i--) counts down from 0, never reaching 10).

🧠 Multiple Choice Questions

1. Which loop is guaranteed to execute its body at least once?

  • A) for loop
  • B) while loop
  • C) do-while loop
  • D) foreach loop
Show Answer

✅ C) do-while loop — The condition is checked AFTER the body executes, so the body always runs at least once.

2. What does break do inside a loop?

  • A) Pauses the loop
  • B) Skips to the next iteration
  • C) Exits the loop immediately
  • D) Restarts the loop
Show Answer

✅ C) Exits the loop immediately — break terminates the innermost loop and continues after it.

3. What does continue do inside a loop?

  • A) Exits the loop
  • B) Skips the rest of the current iteration and goes to the next
  • C) Restarts the program
  • D) Pauses execution
Show Answer

✅ B) Skips the rest of the current iteration and goes to the next — continue jumps to the loop's update expression (for) or condition check (while).

4. What is the output of: for (int i = 0; i < 3; i++) { System.out.print(i + " "); }?

  • A) 1 2 3
  • B) 0 1 2
  • C) 0 1 2 3
  • D) 1 2
Show Answer

✅ B) 0 1 2 — Loop starts at i=0, runs while i<3 (i=0,1,2), increments after each iteration.
