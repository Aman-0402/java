Chapter 14: Inheritance

Inheritance lets one class automatically gain all the fields and methods of another, so you can build specialised classes without rewriting shared code.

The "Is-A" Relationship

Think about animals. A dog is an animal. A cat is an animal. Every animal breathes and eats. Rather than adding breathing and eating to both the Dog class and the Cat class separately, you can put those behaviours in a single Animal class and let Dog and Cat inherit them automatically.

This is called the is-a relationship: a dog is a (kind of) animal, so Dog can inherit from Animal.

The extends Keyword

In Java, inheritance is declared with the extends keyword. The class that is inherited from is called the parent (or superclass), and the class that inherits is called the child (or subclass). 

// Parent class
public class Animal {
    String name;

    public void eat() {
        System.out.println(name + " is eating.");
    }

    public void breathe() {
        System.out.println(name + " is breathing.");
    }
}

// Child class — inherits everything from Animal
public class Dog extends Animal {
    String breed;

    public void bark() {
        System.out.println(name + " says: Woof!");
    }
}

Dog now has name, eat(), breathe(), and bark() — even though only bark() was written in Dog.

Calling the Parent Constructor with super()

When you define a constructor in a child class, you often need to call the parent's constructor first to set up the inherited fields. You do this with super(), which must be the first line in the child constructor. 

public class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " is eating.");
    }
}

public class Dog extends Animal {
    String breed;

    public Dog(String name, String breed) {
        super(name);          // calls Animal(String name) first
        this.breed = breed;
    }

    public void bark() {
        System.out.println(name + " (" + breed + ") says: Woof!");
    }

    public static void main(String[] args) {
        Dog d = new Dog("Rex", "Labrador");
        d.eat();    // inherited from Animal
        d.bark();   // defined in Dog
    }
}

Output:

Rex is eating.
Rex (Labrador) says: Woof!

Overriding a Parent Method with @Override

Sometimes the child class wants to replace the parent's version of a method with its own. This is called method overriding. Place @Override above the method to tell Java (and readers) that this is intentional. 

public class Animal {
    public void speak() {
        System.out.println("...");
    }
}

public class Dog extends Animal {
    @Override
    public void speak() {
        System.out.println("Woof!");
    }
}

public class Cat extends Animal {
    @Override
    public void speak() {
        System.out.println("Meow!");
    }
}

Calling the Parent Method with super.method()

If you override a method but still want to run the parent's version inside it, use super.methodName(): 

public class Dog extends Animal {
    @Override
    public void speak() {
        super.speak();           // prints "..."
        System.out.println("Woof!");
    }
}
💡 Tip: Always add @Override when you intend to override a method. If you misspell the method name, the annotation causes a compile error — without it, Java would silently create a new, unrelated method instead.
⚠️ Common mistake: Java supports only single inheritance — a class can extend only one parent class. If you need behaviour from multiple sources, use interfaces (Chapter 17).
ℹ️ Every class in Java automatically extends Object (from java.lang), even if you don't write extends. That is why every object has methods like toString() and equals() available for free.
