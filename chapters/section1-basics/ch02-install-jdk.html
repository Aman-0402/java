📗 Section 1 — Java Basics

Chapter 2: Install JDK 21

Set up your Java development environment from scratch — after this chapter you can compile and run Java programs on your computer.

What is JDK?

JDK stands for Java Development Kit. It is the complete toolkit you need to write Java programs. It contains:

  • javac — the compiler that turns your .java files into bytecode
  • java — the runtime that runs your compiled programs
  • Standard libraries — thousands of built-in tools Java gives you for free
💡 Always install the JDK, not just the JRE. JRE (Java Runtime Environment) can only run programs — it cannot compile them. You need both compiler and runtime to develop Java. The JDK includes both.

Which Version to Install?

Install JDK 21. It is the current LTS (Long-Term Support) version — it receives security updates for years and is used in production by most companies. Avoid non-LTS versions for learning.

ℹ️ LTS versions: 8, 11, 17, 21. Always pick the latest LTS. At the time of writing, that is JDK 21.

Step 1: Download JDK 21

  1. Go to adoptium.net — this is Eclipse Temurin, a free and open-source JDK.
  2. Select: Version 21 (LTS)
  3. Your OS should auto-detect. Click Download Temurin 21.
ℹ️ Adoptium (Temurin) is recommended — it is free, open source, and has no commercial licence restrictions. Oracle's JDK is also fine but has a commercial licence for production use.

Step 2: Install

Windows: Run the .msi installer. Follow the wizard. When asked, check "Set JAVA_HOME variable" if that option appears — it saves you a manual step later.

Mac: Run the .pkg installer. Follow the wizard. Done.

Linux (Ubuntu/Debian): Run these commands in your terminal:

sudo apt update
sudo apt install temurin-21-jdk

Step 3: Verify the Installation

Open Terminal (Mac/Linux) or Command Prompt / PowerShell (Windows) and run:

java -version

You should see something like:

openjdk version "21.0.3" 2024-04-16
OpenJDK Runtime Environment Temurin-21.0.3+9

Also verify the compiler:

javac -version

Expected output:

javac 21.0.3
⚠️ If you get "java is not recognized" or "command not found", Java is installed but not in your system PATH. See the fix below.

Fixing PATH on Windows

If java -version fails on Windows:

  1. Search for "Environment Variables" in the Start menu → click Edit the system environment variables
  2. Click Environment Variables button
  3. Under System variables, find Path → click Edit
  4. Click New → add the path to your JDK's bin folder, for example: C:\Program Files\Eclipse Adoptium\jdk-21\bin
  5. Click OK on all dialogs, then restart your terminal

Set JAVA_HOME (Recommended)

Some tools like Maven and Spring Boot need the JAVA_HOME environment variable to find your JDK.

Windows: In Environment Variables, add a new System variable: Name = JAVA_HOME, Value = C:\Program Files\Eclipse Adoptium\jdk-21 (no \bin at the end).

Mac/Linux: Add these lines to your ~/.bashrc or ~/.zshrc file:

export JAVA_HOME=$(/usr/libexec/java_home -v 21)
export PATH=$JAVA_HOME/bin:$PATH

Then run source ~/.zshrc (or ~/.bashrc) to apply the changes.

✅ Try It

Open your terminal and run java -version. Confirm you see version 21. Then run javac -version. Both commands must succeed before moving to the next chapter. If either fails, fix the PATH and try again.
