📗 Section 1 — Java Basics

Chapter 20: String Manipulation

Strings are immutable sequences of characters in Java — understanding their methods and the StringBuilder alternative makes you far more effective with text processing.

Strings Are Immutable

Imagine writing on a sticky note. Once you write something, you can't erase it — you can only throw it away and write a new note. Java String objects work the same way: once created, the content cannot change. Any operation that seems to modify a string actually creates a brand-new string and returns it. 

String greeting = "Hello";
greeting.toUpperCase();           // returns "HELLO" but greeting is UNCHANGED
System.out.println(greeting);     // still prints: Hello

greeting = greeting.toUpperCase(); // NOW greeting points to the new string
System.out.println(greeting);      // HELLO
⚠️ Common mistake: Calling a String method and ignoring the return value. String methods never modify the original — they always return a new string. Always assign the result back if you need it.
🔍 Fun Fact: In Java, Strings are immutable — once created, they cannot be changed. When you write str = str + " world", Java creates a brand-new String object and discards the old one. This is why using += in a loop to build a String is slow — it creates thousands of temporary objects. Always use StringBuilder in loops.

Essential String Methods

public class StringDemo {
    public static void main(String[] args) {
        String s = "  Hello, World!  ";

        // Length
        System.out.println(s.length());          // 18 (spaces count)

        // Trim whitespace
        String trimmed = s.trim();
        System.out.println(trimmed);             // "Hello, World!"

        // Case conversion
        System.out.println(trimmed.toUpperCase()); // HELLO, WORLD!
        System.out.println(trimmed.toLowerCase()); // hello, world!

        // Character at index (0-based)
        System.out.println(trimmed.charAt(0));   // H

        // Substring
        System.out.println(trimmed.substring(7));     // World!
        System.out.println(trimmed.substring(7, 12)); // World

        // Find index of a character or substring (-1 if not found)
        System.out.println(trimmed.indexOf("o"));    // 4 (first 'o')
        System.out.println(trimmed.lastIndexOf("o")); // 8

        // Check if contains a sequence
        System.out.println(trimmed.contains("World")); // true

        // Replace
        System.out.println(trimmed.replace("World", "Java")); // Hello, Java!

        // Split into array
        String csv = "Alice,Bob,Carol";
        String[] names = csv.split(",");
        for (String name : names) {
            System.out.println(name);  // Alice / Bob / Carol
        }

        // Check start/end
        System.out.println(trimmed.startsWith("Hello")); // true
        System.out.println(trimmed.endsWith("!"));       // true

        // Convert to char array
        char[] chars = "Java".toCharArray();
        System.out.println(chars[0]); // J
    }
}

equals() vs == for Strings

This is one of the most common Java traps. The == operator checks whether two variables point to the exact same object in memory. The .equals() method checks whether two strings contain the same sequence of characters. 

String a = new String("hello");
String b = new String("hello");

System.out.println(a == b);       // false — different objects in memory
System.out.println(a.equals(b));  // true  — same characters
💡 Tip: Always use .equals() (or .equalsIgnoreCase()) to compare string content. Reserve == for primitives like int and boolean. When comparing to a known literal, put the literal first to avoid a NullPointerException: "hello".equals(userInput).

String Concatenation and the + Operator

You can join strings with +. This is easy to read, but has a hidden cost: because strings are immutable, each + inside a loop creates a new String object and discards the old one. With thousands of iterations, this produces a lot of garbage. 

// Inefficient — creates 1000 intermediate String objects
String result = "";
for (int i = 0; i < 1000; i++) {
    result += "a";   // each += allocates a new String
}
🌍 Real World: Google processes over 8.5 billion searches per day. Each search query is a String that gets parsed, tokenised, and analysed. String manipulation — splitting by spaces, lowercasing, removing punctuation — is at the very core of Google's search pipeline. Efficient string handling is not an academic exercise.

StringBuilder — Mutable String Building

StringBuilder is a mutable character buffer. You append to the same object without creating new String objects every time. For building strings in loops, always prefer StringBuilder. 

public class Main {
    public static void main(String[] args) {
        // Efficient — one object, append in place
        StringBuilder sb = new StringBuilder();

        for (int i = 1; i <= 5; i++) {
            sb.append("Item ").append(i).append("\n");
        }

        sb.insert(0, "--- List ---\n");  // insert at position 0
        sb.delete(0, 13);                // delete "--- List ---\n"
        sb.reverse();                    // reverse the whole content

        System.out.println(sb.toString());  // convert back to String when done

        // Common StringBuilder methods
        StringBuilder name = new StringBuilder("Hello");
        name.append(", World");
        name.replace(0, 5, "Goodbye");
        System.out.println(name);           // Goodbye, World
        System.out.println(name.length());  // 13
    }
}

Useful String Conversions

// int to String
int num = 42;
String s1 = String.valueOf(num);    // "42"
String s2 = Integer.toString(num);  // "42"
String s3 = "" + num;               // "42" (quick but inefficient)

// String to int
String text = "123";
int parsed = Integer.parseInt(text);  // 123

// String to double
double d = Double.parseDouble("3.14"); // 3.14

// Check if empty or blank
String empty = "";
System.out.println(empty.isEmpty());   // true
String blank = "   ";
System.out.println(blank.isBlank());   // true (Java 11+)
ℹ️ String.format() (or the newer formatted() in Java 15+) lets you build strings with placeholders — similar to printf in other languages: String msg = String.format("Hello, %s! You are %d years old.", name, age);
📜 History: String immutability in Java was a deliberate security decision. If Strings were mutable, a database connection URL (a String) could be modified after security validation but before use — a classic TOCTOU (Time-of-Check to Time-of-Use) attack. Immutability prevents this entire class of security vulnerabilities.
