📗 Section 1 — Java Basics

Chapter 17: Interfaces

An interface is a contract — it specifies what methods a class must provide, without dictating how those methods work.

What Is an Interface?

Imagine a job application form. The company doesn't care what university you attended or how you learned your skills — they just need to know you can do certain things: "Can you write code? Can you communicate in English? Can you work in a team?" That list of requirements is the contract.

In Java, an interface is that contract. It lists method names and signatures that a class promises to implement. Any class that signs the contract (uses implements) is guaranteed to have all those methods.

Declaring an Interface

Use the interface keyword. Methods inside an interface are implicitly public abstract — you don't need to write those keywords yourself. 

public interface Flyable {
    void fly();        // implicitly public abstract
    int getAltitude(); // every Flyable thing must be able to report altitude
}

Implementing an Interface

A class "signs the contract" using the implements keyword. It must provide a body for every method listed in the interface, or the compiler will refuse. 

public class Airplane implements Flyable {
    private int altitude;

    @Override
    public void fly() {
        altitude = 35000;
        System.out.println("Airplane is flying at " + altitude + " feet.");
    }

    @Override
    public int getAltitude() {
        return altitude;
    }
}

Implementing Multiple Interfaces

This is where interfaces shine over abstract classes. Java classes can only extend one parent, but they can implement as many interfaces as needed. A Duck, for example, can both fly and swim: 

public interface Flyable {
    void fly();
}

public interface Swimmable {
    void swim();
}

// Duck signs BOTH contracts
public class Duck implements Flyable, Swimmable {
    private String name;

    public Duck(String name) {
        this.name = name;
    }

    @Override
    public void fly() {
        System.out.println(name + " flaps its wings and glides.");
    }

    @Override
    public void swim() {
        System.out.println(name + " paddles across the pond.");
    }

    public static void main(String[] args) {
        Duck d = new Duck("Donald");
        d.fly();
        d.swim();

        // Can also be referenced by either interface type
        Flyable  f = new Duck("Daffy");
        Swimmable s = new Duck("Scrooge");
        f.fly();
        s.swim();
    }
}

Output:

Donald flaps its wings and glides.
Donald paddles across the pond.
Daffy flaps its wings and glides.
Scrooge paddles across the pond.

Default Methods (Java 8+)

Before Java 8, interfaces could only have abstract methods. Java 8 added default methods — methods with a body inside the interface. This lets you add new behaviour to an interface without breaking existing classes that already implement it.

🔍 Fun Fact: Java 8 (2014) added default methods to interfaces — a significant shift. Before Java 8, interfaces could only have abstract methods. The reason for the change: Java needed to add stream-related methods to List and Collection, but couldn't break the millions of existing classes that implemented them. Default methods were the elegant solution.
public interface Greetable {
    String getName();

    // Default method — classes get this for free, can override if needed
    default void greet() {
        System.out.println("Hello, I am " + getName() + "!");
    }
}

public class Robot implements Greetable {
    private String name;

    public Robot(String name) { this.name = name; }

    @Override
    public String getName() { return name; }
    // greet() is inherited from the interface for free
}

Interface vs Abstract Class — When to Use Which

Feature Interface Abstract Class
Multiple inheritance ✅ Yes (implement many) ❌ No (extend one only)
Fields Constants only Any fields
Constructors ❌ None ✅ Yes
Use when Defining a capability/role Sharing base code among related classes
💡 Tip: Favour interfaces over abstract classes when you are defining a capability (something a class can do, like fly or serialize). Use abstract classes when you want to share common state or implementation between closely related classes.
🌍 Real World: The Comparable interface (with its single compareTo() method) is used internally by Arrays.sort() and Collections.sort(). Implement Comparable on your class and Java's entire sorting infrastructure works with it instantly — the same pattern used in Amazon's product rankings and Netflix's content ordering.
📜 History: Interfaces in Java were inspired by "protocols" in Objective-C (1984) and "pure abstract classes" in C++. James Gosling saw them as the clean alternative to multiple inheritance. The decision to separate implements (interfaces) from extends (inheritance) was one of Java's most influential design choices.
⚠️ Common mistake: Forgetting to implement all methods of an interface. The compiler will tell you exactly which methods are missing — read the error message carefully.
