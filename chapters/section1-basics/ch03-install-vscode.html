📗 Section 1 — Java Basics

Chapter 3: Setting Up VS Code for Java

Learn how to install Visual Studio Code, set up the Java Extension Pack, and run your very first Java program — all without touching the command line (yet).

Why VS Code?

Think of VS Code like a smart notebook for writing code. A plain text editor (like Notepad) lets you write words, but VS Code understands Java — it highlights mistakes, completes your sentences, and even has a button to run your program. It is free, lightweight, and used by millions of developers worldwide.

ℹ️ VS Code is not the only option. You could also use IntelliJ IDEA (very popular for Java) or Eclipse. But VS Code is the easiest to set up for a beginner starting today.
🔍 Fun Fact: VS Code was released by Microsoft in 2015 and became the most-used editor in the world within just 3 years. In the 2023 Stack Overflow Developer Survey, 73% of developers use VS Code — making it one of the fastest-adopted developer tools in history.
📜 History: Before VS Code, Java developers used Eclipse (released 2001) or IntelliJ IDEA (also 2001). Both are still popular in enterprise companies today. VS Code took over because of its speed, open-source licensing, and massive extension ecosystem.

Step 1 — Download and Install VS Code

Go to code.visualstudio.com and click the big download button for your operating system (Windows, macOS, or Linux). Run the installer and follow the on-screen steps. When it asks if you want to "Add to PATH", check that box — it lets you open VS Code from the terminal later.

💡 On Windows, also check "Add Open with Code action to Windows Explorer context menu" — it lets you right-click any folder and open it directly in VS Code.

Step 2 — Install the Java Extension Pack

VS Code by itself does not know anything about Java. You need to install an extension — a plug-in that teaches VS Code how Java works.

  1. Open VS Code.
  2. Click the Extensions icon on the left sidebar (it looks like four small squares).
  3. In the search box, type Extension Pack for Java.
  4. Click Install on the result published by Microsoft.

This one pack installs six tools at once: language support, a debugger, Maven and Gradle tools, a test runner, and a project manager. You do not need to understand all of them right now — they will just work in the background.

💡 After installing, VS Code may ask you to reload the window. Click "Reload" to activate the extension.

Step 3 — Open a Folder (Your Workspace)

VS Code works best when you open an entire folder, not a single file. Think of a folder as your project's desk — everything related to your project lives there.

  1. Create a new folder somewhere on your computer, e.g. MyJavaProjects on your Desktop.
  2. In VS Code, go to File → Open Folder… and select that folder.
  3. You will see the folder name appear in the Explorer panel on the left.

Step 4 — Create Your First .java File

In the Explorer panel, click the New File icon (a page with a plus sign). Name the file HelloWorld.java. The .java extension tells VS Code this is a Java file, so it will activate syntax highlighting and IntelliSense (auto-complete) immediately.

⚠️ The filename matters! In Java, the filename must exactly match the class name inside it, including capital letters. Name the file HelloWorld.java, not helloworld.java or hello_world.java.

Step 5 — Write and Run Code

Type this code into your new file:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello from VS Code!");
    }
}

Now you have two ways to run it:

Option A — The Run Button (easiest)

Look for a small ▶ Run link that appears just above the main method after the extension loads. Click it. VS Code will compile and run your program, and the output appears in the Terminal panel at the bottom.

Option B — The Terminal

Go to Terminal → New Terminal. Two commands do the job:

// In the terminal (these are shell commands, not Java):
// Step 1 — Compile (turn .java into .class)
javac HelloWorld.java

// Step 2 — Run the compiled class
java HelloWorld

You should see Hello from VS Code! printed in the terminal.

ℹ️ The Run button actually runs the same two commands behind the scenes. Learning the terminal version is useful because it teaches you what really happens — VS Code is just automating it for convenience.

Useful Extra Extensions

Once you are comfortable, these extensions make life easier:

  • Prettier — automatically formats your code so it looks neat.
  • GitLens — shows who changed which line and when (useful when learning Git).
  • Error Lens — shows error messages inline right next to the broken code instead of making you hover over the red squiggle.
  • Bracket Pair Colorizer 2 — colours matching {} brackets in pairs so you always know which one closes which.
💡 Don't install dozens of extensions at once. Add one at a time, use it for a week, and decide if it helps you. Too many extensions can slow VS Code down.
🌍 Real World: The Java Extension Pack for VS Code has been downloaded over 30 million times. It includes IntelliSense (AI-powered code completion), a built-in debugger, and a test runner — all completely free. Professional developers at Microsoft, Google, and Amazon use the exact same tool you just installed.
