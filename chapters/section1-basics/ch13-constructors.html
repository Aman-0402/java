📗 Section 1 — Java Basics

Chapter 13: Constructors

A constructor is a special method that runs automatically the moment an object is created, letting you set it up with initial data right away.

The Problem Without Constructors

In the last chapter you created a Car object and then set each field individually: 

Car car1 = new Car();
car1.make  = "Toyota";
car1.model = "Corolla";
car1.year  = 2022;

That is four separate lines just to get a usable car. If you forgot to set a field, the car would have null for its make — a recipe for bugs. A constructor solves this by letting you supply the data at the very moment you call new.

What Is a Constructor?

Think of a constructor like a factory order form. When you submit the form (call new), the factory already knows every detail and produces a fully configured product. No follow-up steps needed.

Technically, a constructor is a special method that:

  • Has exactly the same name as the class.
  • Has no return type — not even void.
  • Runs automatically whenever you write new ClassName(...).

Default Constructor

If you write no constructor at all, Java silently provides a free default constructor that takes no arguments and does nothing. That is why new Car() worked in Chapter 12 even though we never wrote a constructor.

ℹ️ The moment you write any constructor yourself, Java's free default constructor disappears. If you still want a no-argument constructor alongside your custom one, you must add it explicitly.
🔍 Fun Fact: Java automatically provides a default (no-argument) constructor if you don't write one. But the moment you write ANY constructor, Java removes the default. This trips up beginners constantly — adding a parameterised constructor and then getting errors because the no-arg constructor silently disappeared.

Parameterized Constructor

A parameterized constructor accepts arguments and uses them to initialise the object's fields: 

public class Person {
    String name;
    int    age;

    // Parameterized constructor
    public Person(String name, int age) {
        this.name = name;   // 'this.name' = field; 'name' = parameter
        this.age  = age;
    }
}

The this Keyword

When a parameter has the same name as a field, Java needs a way to tell them apart. this.name means "the field that belongs to this object", while plain name means the parameter. this is simply a reference to the current object instance.

💡 Tip: Using the same name for both the parameter and the field (like name above) is standard Java style. Always prefix the field assignment with this. inside a constructor — it prevents subtle mix-up bugs.
📜 History: The keyword new for creating objects comes from Simula (1967), was adopted by C++ (1985), and then Java (1995). new explicitly allocates memory on the heap — making it one of the most important keywords in all of object-oriented programming.
⚠️ Common mistake: Writing name = name; without this assigns the parameter to itself and leaves the field unchanged — a silent bug that the compiler will not catch.

Constructor Overloading

You can have multiple constructors in the same class as long as each has a different parameter list. Java picks the right one based on what you pass to new. 

public class Person {
    String name;
    int    age;

    // No-arg constructor
    public Person() {
        this.name = "Unknown";
        this.age  = 0;
    }

    // Name only
    public Person(String name) {
        this.name = name;
        this.age  = 0;
    }

    // Name and age
    public Person(String name, int age) {
        this.name = name;
        this.age  = age;
    }

    public static void main(String[] args) {
        Person p1 = new Person();
        Person p2 = new Person("Alice");
        Person p3 = new Person("Bob", 30);

        System.out.println(p1.name + " — " + p1.age);
        System.out.println(p2.name + " — " + p2.age);
        System.out.println(p3.name + " — " + p3.age);
    }
}

Output:

Unknown — 0
Alice — 0
Bob — 30
🌍 Real World: When you open Instagram, thousands of Java objects are constructed every second on their servers. Each API request creates new User, Session, and Request objects. Efficient constructors matter at scale — Instagram handles over 2 billion users.

🧠 Multiple Choice Questions

1. What is a constructor?

  • A) A special method that returns a new object
  • B) A method called automatically when an object is created to initialize it
  • C) A method that copies an object
  • D) A method that destroys an object
Show Answer

✅ B) A method called automatically when an object is created to initialize it — Constructors have the same name as the class and no return type.

2. If you don't write any constructor, Java provides:

  • A) No constructor (compile error if you use new)
  • B) A default no-arg constructor
  • C) A constructor with all fields as parameters
  • D) A static factory method
Show Answer

✅ B) A default no-arg constructor — Java automatically provides a default constructor ONLY if you haven't defined any constructor.

3. What is a copy constructor?

  • A) A constructor that copies memory
  • B) A constructor that takes another object of the same class and copies its values
  • C) A constructor that duplicates the class
  • D) A built-in Java feature
Show Answer

✅ B) A constructor that takes another object of the same class and copies its values — Java doesn't provide copy constructors automatically (unlike C++); you write them manually.

4. Can a constructor call another constructor in the same class?

  • A) No, never
  • B) Yes, using super()
  • C) Yes, using this() as the first statement
  • D) Yes, using new()
Show Answer

✅ C) Yes, using this() as the first statement — Constructor chaining with this() must be the very first line in the constructor.
