📗 Section 1 — Java Basics

Chapter 13: Constructors

A constructor is a special method that runs automatically the moment an object is created, letting you set it up with initial data right away.

The Problem Without Constructors

In the last chapter you created a Car object and then set each field individually: 

Car car1 = new Car();
car1.make  = "Toyota";
car1.model = "Corolla";
car1.year  = 2022;

That is four separate lines just to get a usable car. If you forgot to set a field, the car would have null for its make — a recipe for bugs. A constructor solves this by letting you supply the data at the very moment you call new.

What Is a Constructor?

Think of a constructor like a factory order form. When you submit the form (call new), the factory already knows every detail and produces a fully configured product. No follow-up steps needed.

Technically, a constructor is a special method that:

  • Has exactly the same name as the class.
  • Has no return type — not even void.
  • Runs automatically whenever you write new ClassName(...).

Default Constructor

If you write no constructor at all, Java silently provides a free default constructor that takes no arguments and does nothing. That is why new Car() worked in Chapter 12 even though we never wrote a constructor.

ℹ️ The moment you write any constructor yourself, Java's free default constructor disappears. If you still want a no-argument constructor alongside your custom one, you must add it explicitly.

Parameterized Constructor

A parameterized constructor accepts arguments and uses them to initialise the object's fields: 

public class Person {
    String name;
    int    age;

    // Parameterized constructor
    public Person(String name, int age) {
        this.name = name;   // 'this.name' = field; 'name' = parameter
        this.age  = age;
    }
}

The this Keyword

When a parameter has the same name as a field, Java needs a way to tell them apart. this.name means "the field that belongs to this object", while plain name means the parameter. this is simply a reference to the current object instance.

💡 Tip: Using the same name for both the parameter and the field (like name above) is standard Java style. Always prefix the field assignment with this. inside a constructor — it prevents subtle mix-up bugs.
⚠️ Common mistake: Writing name = name; without this assigns the parameter to itself and leaves the field unchanged — a silent bug that the compiler will not catch.

Constructor Overloading

You can have multiple constructors in the same class as long as each has a different parameter list. Java picks the right one based on what you pass to new. 

public class Person {
    String name;
    int    age;

    // No-arg constructor
    public Person() {
        this.name = "Unknown";
        this.age  = 0;
    }

    // Name only
    public Person(String name) {
        this.name = name;
        this.age  = 0;
    }

    // Name and age
    public Person(String name, int age) {
        this.name = name;
        this.age  = age;
    }

    public static void main(String[] args) {
        Person p1 = new Person();
        Person p2 = new Person("Alice");
        Person p3 = new Person("Bob", 30);

        System.out.println(p1.name + " — " + p1.age);
        System.out.println(p2.name + " — " + p2.age);
        System.out.println(p3.name + " — " + p3.age);
    }
}

Output:

Unknown — 0
Alice — 0
Bob — 30
