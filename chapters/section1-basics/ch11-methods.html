📗 Section 1 — Java Basics

Chapter 11: Methods

Package reusable actions into named methods so you can call them anytime, avoid repeating code, and keep your programs clean and organised.

What is a Method?

A method is a named block of code that performs one specific action. Think of it like a recipe card: you write the instructions once on a card ("How to make toast"), and then anyone can follow that card whenever they need toast — without re-writing the steps each time. In Java, you write the method once, then call it as many times as you need.

You have been using methods all along: System.out.println() is a method. scanner.nextLine() is a method. Now you will write your own.

Declaring a Method

A method declaration has this structure:

// returnType methodName(parameterType parameterName, ...) {
//     // body — what the method does
//     return value; // only needed if returnType is not void
// }
public class MethodBasics {

    // A method that prints a greeting — returns nothing (void)
    static void greet() {
        System.out.println("Hello! Welcome to Java.");
    }

    public static void main(String[] args) {
        greet();   // calling the method
        greet();   // calling it again — reuse!
        greet();   // one more time
    }
}

🔍 How This Code Works

  • static void greet() — declares a method that belongs to the class (static), returns nothing (void), and is named greet
  • System.out.println(...) — the single action this method performs every time it is called
  • greet(); — calling the method by name followed by parentheses; the parentheses are required even when passing no arguments
  • greet(); greet(); greet(); — the same method is called three times without rewriting its body, demonstrating the core benefit of methods: code reuse

Return Types

A method can either return a value back to the caller, or return nothing. The return type is declared before the method name:

  • void — the method does something (like print) but does not return a value.
  • int, double, String, etc. — the method computes something and sends a result back.
public class ReturnTypeDemo {

    // Returns nothing — just prints
    static void printDivider() {
        System.out.println("--------------------");
    }

    // Returns the area of a rectangle
    static double calculateArea(double width, double height) {
        double area = width * height;
        return area;  // send the result back to whoever called this method
    }

    // Returns a greeting string
    static String buildGreeting(String name) {
        return "Hello, " + name + "!";
    }

    public static void main(String[] args) {
        printDivider();

        double roomArea = calculateArea(5.5, 4.0);
        System.out.println("Room area: " + roomArea + " sq metres");

        String msg = buildGreeting("Alice");
        System.out.println(msg);

        printDivider();
    }
}

🔍 How This Code Works

  • static void printDivider() — a void method; it performs an action (printing) but gives nothing back to the caller
  • static double calculateArea(double width, double height) — declares that this method will compute and hand back a double value
  • return area; — sends the computed result back to the call site; execution of the method stops here
  • double roomArea = calculateArea(5.5, 4.0); — the returned value is captured in a variable for later use
  • static String buildGreeting(String name) — shows that return types are not limited to numbers; any type, including String, can be returned
  • return "Hello, " + name + "!"; — a return statement can contain an expression; Java evaluates it first, then returns the result
💡 Once a return statement executes, the method immediately stops. Any code after return in the same block is unreachable and will cause a compiler warning. Use this intentionally to exit a method early when needed.
📜 History: The concept of "subroutines" — what we now call methods or functions — was invented by Alan Turing in 1945. He called them "abbreviated code definitions." They've been central to programming for over 80 years, predating almost every modern language feature.

Parameters and Arguments

Parameters are the variable names listed inside the method's parentheses when you define it. Arguments are the actual values you pass when you call it. The terms are often used interchangeably, but the distinction helps when reading error messages.

public class ParamVsArg {

    // width and height are PARAMETERS (placeholders defined with the method)
    static double calculateArea(double width, double height) {
        return width * height;
    }

    public static void main(String[] args) {
        // 5.5 and 3.0 are ARGUMENTS (real values passed when calling)
        double area = calculateArea(5.5, 3.0);
        System.out.println("Area: " + area);  // Area: 16.5

        // You can pass variables as arguments too
        double w = 8.0;
        double h = 2.5;
        System.out.println("Area: " + calculateArea(w, h));  // Area: 20.0
    }
}

🔍 How This Code Works

  • static double calculateArea(double width, double height) — width and height are parameters: named placeholders defined in the method signature
  • calculateArea(5.5, 3.0) — 5.5 and 3.0 are the arguments: the actual values supplied at the call site
  • double area = calculateArea(5.5, 3.0); — the returned double is stored immediately in a variable
  • double w = 8.0; calculateArea(w, h) — variables can also be passed as arguments; their current values are copied into the parameters

Method Overloading

Java allows multiple methods with the same name, as long as they have different parameter lists. This is called overloading. Java picks the right version based on what arguments you pass.

public class OverloadDemo {

    // Version 1: two integers
    static int add(int a, int b) {
        return a + b;
    }

    // Version 2: three integers
    static int add(int a, int b, int c) {
        return a + b + c;
    }

    // Version 3: two doubles
    static double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        System.out.println(add(3, 4));          // calls version 1 -> 7
        System.out.println(add(1, 2, 3));       // calls version 2 -> 6
        System.out.println(add(1.5, 2.5));      // calls version 3 -> 4.0
    }
}

🔍 How This Code Works

  • static int add(int a, int b) / static int add(int a, int b, int c) / static double add(double a, double b) — three methods share the same name; Java distinguishes them by parameter count and types
  • add(3, 4) — Java matches two int arguments to version 1 at compile time
  • add(1, 2, 3) — three int arguments → version 2
  • add(1.5, 2.5) — two double arguments → version 3; note the return type alone cannot distinguish overloads
  • Method selection happens at compile time based on the argument types, not at runtime
ℹ️ Overloading is different from having different method names. addInts and addDoubles would also work — but overloading with the same name add is cleaner and more intuitive for the caller.
🌍 Real World: Every time you click "Like" on Instagram, a method called something like likePost(userId, postId) runs on Instagram's servers. That single method triggers cascading calls: update database, send notification, update feed ranking — all separate methods working together.

Static vs Instance Methods (Brief)

You have been writing static methods. A static method belongs to the class itself — you can call it without creating an object. An instance method (no static keyword) belongs to a specific object and can access that object's data. You will explore instance methods deeply in the Object-Oriented chapters. For now, just know:

public class StaticVsInstance {

    // static method — call it using the class name (or just its name inside same class)
    static void staticHello() {
        System.out.println("Hello from a static method!");
    }

    // instance method — would need an object to call
    void instanceHello() {
        System.out.println("Hello from an instance method!");
    }

    public static void main(String[] args) {
        staticHello();  // OK — no object needed

        // To call an instance method from main (which is static), create an object first:
        StaticVsInstance obj = new StaticVsInstance();
        obj.instanceHello();  // OK — calling on an object
    }
}
⚠️ A common error: "non-static method cannot be referenced from a static context." This happens when you try to call an instance method directly from main without creating an object first. Either make the method static, or create an object and call it on that.

Full Example — Rectangle Calculator

public class RectangleCalculator {

    static double area(double width, double height) {
        return width * height;
    }

    static double perimeter(double width, double height) {
        return 2 * (width + height);
    }

    static boolean isSquare(double width, double height) {
        return width == height;
    }

    static void printReport(double width, double height) {
        System.out.println("Rectangle " + width + " x " + height);
        System.out.printf("  Area      : %.2f%n", area(width, height));
        System.out.printf("  Perimeter : %.2f%n", perimeter(width, height));
        System.out.println("  Is square : " + isSquare(width, height));
    }

    public static void main(String[] args) {
        printReport(5.0, 3.0);
        System.out.println();
        printReport(4.0, 4.0);
    }
}

🔍 How This Code Works

  • static double area(double width, double height) — a focused single-purpose helper; doing one thing makes it easy to test in isolation
  • static boolean isSquare(double width, double height) — returns a boolean directly from the comparison expression width == height, no need for an if statement
  • static void printReport(...) — an orchestrator method that calls three other methods; this shows how large tasks decompose into smaller, named steps
  • System.out.printf(" Area : %.2f%n", area(width, height)); — a method call (area(...)) is used directly as an argument to printf; the return value is consumed inline
  • printReport(5.0, 3.0); — calling the report method twice with different values demonstrates how one method replaces repeating an entire block of output logic

Output:

Rectangle 5.0 x 3.0
  Area      : 15.00
  Perimeter : 16.00
  Is square : false

Rectangle 4.0 x 4.0
  Area      : 16.00
  Perimeter : 16.00
  Is square : true
💡 A good method does one thing and does it well. If you find yourself writing a method that does five different things, split it into five smaller methods. Smaller methods are easier to test, easier to understand, and easier to reuse.
🔍 Fun Fact: The longest method ever found in production code had over 66,000 lines. It was discovered at a Fortune 500 company during a code audit. Methods should ideally be under 20 lines — this principle is called the "Single Responsibility Principle."

🧠 Multiple Choice Questions

1. What keyword is used when a method returns nothing?

  • A) null
  • B) empty
  • C) void
  • D) none
Show Answer

✅ C) void — public void printName() signals the method has no return value.

2. What is method overloading?

  • A) Using the same method in a subclass
  • B) Having multiple methods with the same name but different parameters
  • C) Calling a method inside itself
  • D) Making a method run faster
Show Answer

✅ B) Having multiple methods with the same name but different parameters — Java distinguishes overloaded methods by parameter count and types.

3. What happens when a method reaches a return statement?

  • A) The program ends
  • B) Execution jumps to the start of the method
  • C) The method exits and control returns to the caller
  • D) Nothing special happens
Show Answer

✅ C) The method exits and control returns to the caller — The returned value (if any) is passed back to where the method was called.

4. In Java, parameters are passed by:

  • A) Reference always
  • B) Value always
  • C) Reference for primitives, value for objects
  • D) Value for primitives, reference for objects
Show Answer

✅ D) Value for primitives, reference for objects — Primitives pass a copy. Objects pass a copy of the reference (the reference itself, not the object).
