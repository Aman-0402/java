📗 Section 1 — Java Basics

Chapter 11: Methods

Package reusable actions into named methods so you can call them anytime, avoid repeating code, and keep your programs clean and organised.

What is a Method?

A method is a named block of code that performs one specific action. Think of it like a recipe card: you write the instructions once on a card ("How to make toast"), and then anyone can follow that card whenever they need toast — without re-writing the steps each time. In Java, you write the method once, then call it as many times as you need.

You have been using methods all along: System.out.println() is a method. scanner.nextLine() is a method. Now you will write your own.

Declaring a Method

A method declaration has this structure:

// returnType methodName(parameterType parameterName, ...) {
//     // body — what the method does
//     return value; // only needed if returnType is not void
// }
public class MethodBasics {

    // A method that prints a greeting — returns nothing (void)
    static void greet() {
        System.out.println("Hello! Welcome to Java.");
    }

    public static void main(String[] args) {
        greet();   // calling the method
        greet();   // calling it again — reuse!
        greet();   // one more time
    }
}

Return Types

A method can either return a value back to the caller, or return nothing. The return type is declared before the method name:

  • void — the method does something (like print) but does not return a value.
  • int, double, String, etc. — the method computes something and sends a result back.
public class ReturnTypeDemo {

    // Returns nothing — just prints
    static void printDivider() {
        System.out.println("--------------------");
    }

    // Returns the area of a rectangle
    static double calculateArea(double width, double height) {
        double area = width * height;
        return area;  // send the result back to whoever called this method
    }

    // Returns a greeting string
    static String buildGreeting(String name) {
        return "Hello, " + name + "!";
    }

    public static void main(String[] args) {
        printDivider();

        double roomArea = calculateArea(5.5, 4.0);
        System.out.println("Room area: " + roomArea + " sq metres");

        String msg = buildGreeting("Alice");
        System.out.println(msg);

        printDivider();
    }
}
💡 Once a return statement executes, the method immediately stops. Any code after return in the same block is unreachable and will cause a compiler warning. Use this intentionally to exit a method early when needed.

Parameters and Arguments

Parameters are the variable names listed inside the method's parentheses when you define it. Arguments are the actual values you pass when you call it. The terms are often used interchangeably, but the distinction helps when reading error messages.

public class ParamVsArg {

    // width and height are PARAMETERS (placeholders defined with the method)
    static double calculateArea(double width, double height) {
        return width * height;
    }

    public static void main(String[] args) {
        // 5.5 and 3.0 are ARGUMENTS (real values passed when calling)
        double area = calculateArea(5.5, 3.0);
        System.out.println("Area: " + area);  // Area: 16.5

        // You can pass variables as arguments too
        double w = 8.0;
        double h = 2.5;
        System.out.println("Area: " + calculateArea(w, h));  // Area: 20.0
    }
}

Method Overloading

Java allows multiple methods with the same name, as long as they have different parameter lists. This is called overloading. Java picks the right version based on what arguments you pass.

public class OverloadDemo {

    // Version 1: two integers
    static int add(int a, int b) {
        return a + b;
    }

    // Version 2: three integers
    static int add(int a, int b, int c) {
        return a + b + c;
    }

    // Version 3: two doubles
    static double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        System.out.println(add(3, 4));          // calls version 1 -> 7
        System.out.println(add(1, 2, 3));       // calls version 2 -> 6
        System.out.println(add(1.5, 2.5));      // calls version 3 -> 4.0
    }
}
ℹ️ Overloading is different from having different method names. addInts and addDoubles would also work — but overloading with the same name add is cleaner and more intuitive for the caller.

Static vs Instance Methods (Brief)

You have been writing static methods. A static method belongs to the class itself — you can call it without creating an object. An instance method (no static keyword) belongs to a specific object and can access that object's data. You will explore instance methods deeply in the Object-Oriented chapters. For now, just know:

public class StaticVsInstance {

    // static method — call it using the class name (or just its name inside same class)
    static void staticHello() {
        System.out.println("Hello from a static method!");
    }

    // instance method — would need an object to call
    void instanceHello() {
        System.out.println("Hello from an instance method!");
    }

    public static void main(String[] args) {
        staticHello();  // OK — no object needed

        // To call an instance method from main (which is static), create an object first:
        StaticVsInstance obj = new StaticVsInstance();
        obj.instanceHello();  // OK — calling on an object
    }
}
⚠️ A common error: "non-static method cannot be referenced from a static context." This happens when you try to call an instance method directly from main without creating an object first. Either make the method static, or create an object and call it on that.

Full Example — Rectangle Calculator

public class RectangleCalculator {

    static double area(double width, double height) {
        return width * height;
    }

    static double perimeter(double width, double height) {
        return 2 * (width + height);
    }

    static boolean isSquare(double width, double height) {
        return width == height;
    }

    static void printReport(double width, double height) {
        System.out.println("Rectangle " + width + " x " + height);
        System.out.printf("  Area      : %.2f%n", area(width, height));
        System.out.printf("  Perimeter : %.2f%n", perimeter(width, height));
        System.out.println("  Is square : " + isSquare(width, height));
    }

    public static void main(String[] args) {
        printReport(5.0, 3.0);
        System.out.println();
        printReport(4.0, 4.0);
    }
}

Output:

Rectangle 5.0 x 3.0
  Area      : 15.00
  Perimeter : 16.00
  Is square : false

Rectangle 4.0 x 4.0
  Area      : 16.00
  Perimeter : 16.00
  Is square : true
💡 A good method does one thing and does it well. If you find yourself writing a method that does five different things, split it into five smaller methods. Smaller methods are easier to test, easier to understand, and easier to reuse.
