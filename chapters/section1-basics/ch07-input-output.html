Chapter 7: Input and Output

Make your programs interactive by reading text that the user types into the terminal, and displaying results back to the screen.

Output You Already Know

You have been using System.out.println since Chapter 4. Let's quickly recap the output methods before moving to input:

public class OutputDemo {
    public static void main(String[] args) {
        System.out.println("This prints and moves to a new line.");
        System.out.print("This prints ");
        System.out.print("on the same line.\n");   // \n = manual new line
        System.out.printf("Formatted: name=%s, age=%d, score=%.2f%n",
                          "Alice", 22, 95.678);
        // Output: Formatted: name=Alice, age=22, score=95.68
    }
}

🔍 How This Code Works

  • System.out.println — prints text and automatically appends a newline; the cursor moves to the next line
  • System.out.print — prints text with no newline; subsequent output continues on the same line
  • \n inside a string — an escape sequence representing a newline character; manually moves output to the next line
  • System.out.printf — prints a formatted string using format specifiers as placeholders
  • %s — placeholder for a String argument; replaced by "Alice" at runtime
  • %d — placeholder for a decimal integer; replaced by 22
  • %.2f — placeholder for a floating-point number rounded to 2 decimal places; 95.678 becomes 95.68
  • %n — portable newline in printf (preferred over \n inside printf because it adapts to the OS line ending)
ℹ️ printf stands for "print formatted." The placeholders work like this: %s = String, %d = integer, %f = decimal (add .2 before f to show 2 decimal places). %n is a portable newline.
🔍 Fun Fact: System.out is a PrintStream object — one of the oldest classes in Java, unchanged since 1996. When you type System.out.println(), you are using a class that is literally older than Google (founded 1998) and older than the first iPhone (2007).
📜 History: System.in, System.out, and System.err are inspired by Unix's standard streams (stdin, stdout, stderr) from the 1970s. Java inherited this concept directly from Unix operating system design — connecting the world of 1970s systems programming to modern Java applications.

Reading Input — The Scanner Class

Imagine a scanner at a supermarket checkout — you wave the barcode in front of it, and it reads the information. Java's Scanner class works the same way: it reads whatever the user types at the keyboard and hands it to your program.

To use Scanner, you need to do two things:

  1. Import it at the top of your file — this tells Java where to find the Scanner code.
  2. Create a Scanner object attached to System.in (the keyboard).
import java.util.Scanner;  // Step 1: import at the very top of the file

public class ScannerIntro {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);  // Step 2: create the scanner

        System.out.print("Enter your name: ");
        String name = scanner.nextLine();           // reads a full line of text

        System.out.println("Hello, " + name + "!");

        scanner.close(); // good practice: close when done
    }
}

🔍 How This Code Works

  • import java.util.Scanner — tells Java where to find the Scanner class; must appear before the class declaration at the top of the file
  • new Scanner(System.in) — creates a Scanner object; System.in is the standard input stream (the keyboard)
  • scanner.nextLine() — pauses the program and waits for the user to type a full line and press Enter; returns everything typed as a String
  • System.out.print("Enter your name: ") — uses print (not println) so the cursor stays on the same line as the prompt, waiting for user input
  • scanner.close() — releases the underlying input stream resource; always close Scanner when finished, especially when reading files
  • "Hello, " + name + "!" — concatenates the greeting with the user's input to build the response string

Reading Different Data Types

Scanner has different methods for reading different types of data:

import java.util.Scanner;

public class ReadingTypes {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter a whole number: ");
        int number = sc.nextInt();        // reads one integer

        System.out.print("Enter a decimal: ");
        double decimal = sc.nextDouble(); // reads a double

        System.out.print("Enter true or false: ");
        boolean flag = sc.nextBoolean();  // reads a boolean

        System.out.println("You entered: " + number + ", " + decimal + ", " + flag);

        sc.close();
    }
}

nextLine()A full line of text (including spaces)Hello World
next()One word (stops at a space)Hello
nextInt()One integer42
nextDouble()One decimal number3.14

The Scanner Pitfall: nextInt() Followed by nextLine()

This is one of the most common bugs beginners hit. When you call nextInt(), Scanner reads the number but leaves the newline character (the Enter key press) sitting in the input buffer. The very next nextLine() call reads that leftover newline instead of waiting for you to type something new. Your program seems to skip the question entirely.

import java.util.Scanner;

public class ScannerBug {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter your age: ");
        int age = sc.nextInt();

        // BUG: this nextLine() will immediately read the leftover newline
        // from after the age, so the name question gets skipped!
        System.out.print("Enter your name: ");
        String name = sc.nextLine();  // reads "" (empty string) immediately

        System.out.println("Age: " + age + ", Name: '" + name + "'");
        // Output: Age: 25, Name: ''   <-- name is empty!

        sc.close();
    }
}
⚠️ The fix: add an extra sc.nextLine(); immediately after nextInt() (or nextDouble()) to consume the leftover newline, then call nextLine() again to read the actual input.

Complete Example — Name and Age Program

import java.util.Scanner;

public class NameAndAge {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter your age: ");
        int age = sc.nextInt();
        sc.nextLine(); // consume the leftover newline after nextInt()

        System.out.print("Enter your full name: ");
        String name = sc.nextLine(); // now correctly waits for input

        System.out.print("Enter your city: ");
        String city = sc.nextLine();

        System.out.println("-----------------------------");
        System.out.println("Name : " + name);
        System.out.println("Age  : " + age);
        System.out.println("City : " + city);
        System.out.println("In 10 years you will be " + (age + 10));

        sc.close();
    }
}

🔍 How This Code Works

  • sc.nextInt() — reads one integer from the keyboard but leaves the newline character (from pressing Enter) still in the input buffer
  • sc.nextLine() immediately after nextInt() — this extra call consumes the leftover newline so it does not interfere with the next real input
  • The second sc.nextLine() — now that the buffer is clean, this correctly waits for the user to type their full name
  • (age + 10) inside the println — parentheses force arithmetic addition first; without them "be " + age + 10 would concatenate strings, giving "be 2210"
  • "-----------------------------" — a visual separator printed to make the output easier to read; just a string literal
  • This pattern (nextInt → nextLine → nextLine) is the standard fix for the most common Scanner beginner bug

Sample run:

Enter your age: 22
Enter your full name: Alice Johnson
Enter your city: New York
-----------------------------
Name : Alice Johnson
Age  : 22
City : New York
In 10 years you will be 32
💡 Always close() your Scanner when you are done with it. It is good programming habit and prevents resource leaks, especially when reading from files (which you will do later).
🌍 Real World: The Scanner class introduced in Java 5 (2004) simplified input reading dramatically. Before that, developers had to use BufferedReader with InputStreamReader — three lines of code just to read one string. Modern production Java apps rarely use Scanner directly — they use HTTP APIs and JSON parsing — but Scanner is the perfect teaching tool for understanding how input works.

  • A) next()
  • B) nextWord()
  • C) nextLine()
  • D) readLine()

✅ C) nextLine() — next() stops at whitespace. nextLine() reads everything until Enter is pressed.
