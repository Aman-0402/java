What Is React?

React is a JavaScript library for building user interfaces. Developed by Facebook, it changed the way developers think about UIs by introducing a simple idea: the UI is a function of state.

Instead of manually updating the DOM (the browser's internal representation of your page) every time data changes, you describe what the UI should look like for a given state, and React figures out the minimum DOM changes needed. Think of it like this:

  • Old way: Manually call document.getElementById('name').innerText = 'Alice' — like writing raw SQL.
  • React way: Declare <p>{name}</p> and update the variable — like using JPA with auto-generated queries.
📜 History: React was created by Jordan Walke at Facebook in 2011. Facebook's news feed was becoming impossible to maintain — UI state was inconsistent and bugs were everywhere. Walke invented React's core ideas (virtual DOM, one-way data flow) on a weekend and presented it internally. React was open-sourced in 2013. It is now the most popular JavaScript UI library in the world.

Why Use React with Spring Boot?

Spring Boot is excellent at serving data as JSON via REST APIs. React is excellent at consuming that JSON and displaying it beautifully. They are a perfect split: Spring Boot owns the backend (data, business logic, security), React owns the frontend (user interface, interactions, routing). This is called a decoupled full-stack architecture.

Setting Up a React App

You need Node.js installed (download from nodejs.org). Then create a project:

# Option 1: Create React App (classic, beginner-friendly)
npx create-react-app my-app
cd my-app
npm start

# Option 2: Vite (faster, modern — recommended)
npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev

Either option opens a browser at http://localhost:3000 (CRA) or http://localhost:5173 (Vite) with a starter React app. Your Spring Boot backend will run separately on port 8080.

💡 Tip: Use Vite for new projects. It starts in under a second and hot-reloads instantly. Create React App is older and slower, but you'll see it in many tutorials.

JSX — HTML Inside JavaScript

React uses JSX (JavaScript XML) — a syntax that lets you write what looks like HTML directly inside JavaScript functions. It gets transformed into regular JavaScript by a build tool (Babel or esbuild). 

// This is JSX — looks like HTML, but it's JavaScript
function Greeting() {
  const name = "Java Developer";
  return (
    <div className="greeting">
      <h1>Hello, {name}!</h1>
      <p>Welcome to React.</p>
    </div>
  );
}

Two key JSX rules to remember:

  • Use className instead of class (because class is a reserved word in JavaScript).
  • Every JSX expression must return a single root element. Wrap siblings in a <div> or an empty <>...</> fragment.
⚠️ JSX is not HTML. class becomes className, for becomes htmlFor, and all tags must be self-closing where HTML allows it: <img />, <input />, <br />. Forgetting these causes cryptic errors.

Functional Components

A React component is just a JavaScript function that returns JSX. Think of it like a Java method that returns a UI fragment. Components start with a capital letter — this is how React distinguishes them from plain HTML tags. 

// CourseCard.jsx — a reusable component
function CourseCard() {
  return (
    <div className="card">
      <h2>Spring Boot REST APIs</h2>
      <p>Learn to build REST APIs with Java.</p>
      <button>Enroll</button>
    </div>
  );
}

export default CourseCard;
// App.jsx — uses the CourseCard component
import CourseCard from './CourseCard';

function App() {
  return (
    <div>
      <h1>Our Courses</h1>
      <CourseCard />
      <CourseCard />
      <CourseCard />
    </div>
  );
}

Props — Passing Data to Components

Right now CourseCard always shows the same hardcoded data. Props let you pass data into a component, just like passing arguments to a Java method. The receiving component accesses them via a props object (or with destructuring). 

// CourseCard.jsx — accepts props
function CourseCard({ title, description, instructor }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <p>{description}</p>
      <small>By {instructor}</small>
      <button>Enroll</button>
    </div>
  );
}

export default CourseCard;
// App.jsx — passes props to CourseCard
import CourseCard from './CourseCard';

function App() {
  return (
    <div>
      <CourseCard
        title="Spring Boot REST APIs"
        description="Build backends with Java."
        instructor="Alice"
      />
      <CourseCard
        title="React for Beginners"
        description="Build UIs with React."
        instructor="Bob"
      />
    </div>
  );
}
💡 Tip: Props flow downward — from parent to child. A child component never modifies its own props. If you need data to flow up, you pass a callback function as a prop.
🔍 Fun Fact: React's "Virtual DOM" is a lightweight JavaScript copy of the real browser DOM. When state changes, React compares the new Virtual DOM with the old one (called "diffing"), finds the minimum changes needed, and only updates those parts of the real DOM. This makes React fast — but the Virtual DOM itself isn't magic, just smart optimization.

useState — Local State

State is data that can change over time and when it changes, React re-renders the component automatically. The useState hook gives a component its own private variable that React watches.

Think of it like a Java instance variable that, when changed, automatically refreshes the screen.

import { useState } from 'react';

function Counter() {
  // useState(0) returns [currentValue, setterFunction]
  const [count, setCount] = useState(0);

  return (
    <div className="counter">
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <button onClick={() => setCount(count - 1)}>-1</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

The pattern is always the same:

const [stateName, setStateName] = useState(initialValue);
⚠️ Never mutate state directly. Always use the setter function. Writing count = count + 1 will NOT trigger a re-render. You must write setCount(count + 1). This is one of the most common React mistakes for beginners.
🌍 Real World: React powers the UIs of Facebook, Instagram, Airbnb, Netflix, and WhatsApp Web. WhatsApp Web's entire chat interface is a React application. When you send a message, React re-renders only the chat window, not the entire page. The responsiveness you experience is React's reconciliation algorithm at work.

useEffect — Side Effects and API Calls

useEffect runs code after the component renders. It's where you do things that interact with the outside world — like fetching data from your Spring Boot API, setting up a timer, or subscribing to an event.

Think of it like a Spring Boot @PostConstruct method — code that runs automatically after setup.

import { useState, useEffect } from 'react';

function CourseList() {
  const [courses, setCourses] = useState([]);      // start with empty array
  const [loading, setLoading] = useState(true);    // track loading state

  useEffect(() => {
    // This runs AFTER the component first renders
    fetch('http://localhost:8080/api/courses')
      .then(response => response.json())
      .then(data => {
        setCourses(data);
        setLoading(false);
      });
  }, []);  // empty array means "run once on mount"

  if (loading) return <p>Loading courses...</p>;

  return (
    <ul>
      {courses.map(course => (
        <li key={course.id}>{course.title}</li>
      ))}
    </ul>
  );
}

The second argument to useEffect — the dependency array — controls when the effect re-runs:

  • [] — run once when the component mounts (like @PostConstruct)
  • [userId] — re-run whenever userId changes
  • no array — run after every render (usually not what you want)

Full Counter App Example

Here is a complete, minimal React app combining everything above:

// App.jsx
import { useState } from 'react';
import './App.css';

function Counter({ label }) {
  const [count, setCount] = useState(0);

  return (
    <div className="counter-card">
      <h2>{label}</h2>
      <p className="count-display">{count}</p>
      <div className="buttons">
        <button onClick={() => setCount(count - 1)}>−</button>
        <button onClick={() => setCount(0)}>Reset</button>
        <button onClick={() => setCount(count + 1)}>+</button>
      </div>
    </div>
  );
}

function App() {
  return (
    <div className="app">
      <h1>My Counters</h1>
      <Counter label="Java Chapters Read" />
      <Counter label="Spring Boot APIs Built" />
    </div>
  );
}

export default App;

✅ Try It

Create a Vite React app and build the Counter example above. Then extend it:

  1. Add a step prop to Counter — instead of always incrementing by 1, increment by the step value. Call it as <Counter label="..." step={5} />.
  2. Add a <p> that shows "Positive", "Negative", or "Zero" based on the current count value (use a ternary or an if inside the component).
  3. Use useEffect to log the count to the browser console every time it changes. (Hint: put count in the dependency array.)
