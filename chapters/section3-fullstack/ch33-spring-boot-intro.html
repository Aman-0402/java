Chapter 33: Introduction to Spring Boot

Learn what Spring Boot is, how it auto-configures a Java server for you, and how to create your first REST endpoint in under 10 lines of code.

What is Spring Boot?

Before Spring Boot, setting up a Java web server required writing hundreds of lines of XML configuration. You had to configure servlets, map URLs, set up connection pools, and assemble dozens of components by hand. It could take days just to get a "Hello World" running.

Spring Boot flips this around with one principle: convention over configuration. Instead of asking you to configure everything, Spring Boot makes sensible decisions for you by default. You only need to override a setting when the default does not fit your needs.

Think of it like a brand-new smartphone. Out of the box it already has a default ringtone, default wallpaper, and default apps. You can change any of them — but you do not have to before you can make your first call.

📜 History: Spring Framework was created by Rod Johnson in 2003, published in his book "Expert One-on-One J2EE Design and Development." It was a reaction against J2EE (Java Enterprise Edition), which required writing 200+ lines of XML config for a simple database connection. Spring's tagline was "make Java development simple again." Spring Boot (2014) took this further — zero XML config required.

The Main Class: @SpringBootApplication

Every Spring Boot project has one main class. When you generated your project from Spring Initializr it was created for you:

package com.example.todoapi;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication   // Magic annotation — explained below
public class TodoApiApplication {

    public static void main(String[] args) {
        SpringApplication.run(TodoApiApplication.class, args);
    }
}

🔍 How This Code Works

  • @SpringBootApplication — a composite annotation combining @SpringBootConfiguration, @EnableAutoConfiguration, and @ComponentScan
  • @EnableAutoConfiguration — detects libraries on the classpath (e.g. Spring Web) and automatically sets up their infrastructure (e.g. Tomcat server)
  • @ComponentScan — scans this package and all sub-packages for Spring-managed classes like @RestController and @Service
  • SpringApplication.run(TodoApiApplication.class, args) — bootstraps the entire Spring context, starts the embedded server, and loads all auto-configuration
  • public static void main(String[] args) — standard Java entry point; Spring Boot apps start just like any ordinary Java program

The @SpringBootApplication annotation is actually a shortcut for three annotations combined:

  • @SpringBootConfiguration — marks this class as a configuration source.
  • @EnableAutoConfiguration — tells Spring Boot to automatically configure libraries based on what is on the classpath. If it sees Spring Web on the classpath it sets up a web server automatically.
  • @ComponentScan — tells Spring to scan this package and all sub-packages for classes annotated with @Component, @Service, @Repository, @Controller, etc., and register them as Spring-managed beans.
ℹ️ What is a Bean? In Spring, a "bean" is simply an object that Spring creates and manages for you. Instead of writing new MyService(), you annotate a class and Spring creates and injects it wherever it is needed. This is called Dependency Injection and it is one of Spring's core ideas.
🔍 Fun Fact: Spring Boot is used by over 60% of Java enterprise developers. Companies using Spring Boot include Netflix (microservices), Alibaba (e-commerce), and Trivago (travel). Spring Boot's @SpringBootApplication annotation triggers 100+ auto-configuration classes that set up your entire application — database connections, web server, JSON serialization — in under a second.

How Auto-Configuration Works

When your app starts, Spring Boot looks at which libraries are on the classpath and configures itself accordingly:

  • Found spring-boot-starter-web? Start an embedded Tomcat server on port 8080.
  • Found spring-boot-starter-data-jpa and h2? Set up a database connection and create tables automatically.
  • Found spring-security? Lock down all endpoints with a login page by default.

You never had to write code to set up Tomcat. It just happened. That is auto-configuration.

Your First REST Controller

A controller is a class that handles incoming HTTP requests and sends back responses. In Spring Boot, you create one by annotating a class with @RestController.

Create a new file at src/main/java/com/example/todoapi/HelloController.java:

package com.example.todoapi;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController   // Marks this class as a REST controller; every method returns data, not a view
public class HelloController {

    @GetMapping("/hello")   // Map HTTP GET requests to /hello to this method
    public String hello() {
        return "Hello, Spring Boot!";
    }
}

🔍 How This Code Works

  • @RestController — marks this class as a REST controller; combines @Controller + @ResponseBody so every method's return value is written directly to the HTTP response
  • @GetMapping("/hello") — registers this method to handle HTTP GET requests at the path /hello
  • public String hello() — returns a plain string; Spring Boot writes it as the response body with Content-Type: text/plain
  • No XML config needed — Spring Boot's component scan detects @RestController and registers the route automatically
  • The embedded Tomcat server (started by @SpringBootApplication) listens on port 8080 and routes the request to this method

That is all you need. Spring Boot will:

  1. Detect the @RestController annotation during component scanning.
  2. Register /hello as a valid URL on your running server.
  3. When a GET request arrives at /hello, call the hello() method and send its return value as the HTTP response body.

Running the App

# From your project root directory
mvn spring-boot:run

Watch the console output. You will see Spring Boot's banner and eventually:

Started TodoApiApplication in 2.345 seconds (process running for 2.678)

Your server is now running on port 8080.

Testing Your Endpoint

You can test the endpoint in three ways:

Option 1 — Browser: Open http://localhost:8080/hello. You will see the text Hello, Spring Boot! on the page.

Option 2 — curl:

curl http://localhost:8080/hello
# Output: Hello, Spring Boot!

Option 3 — VS Code REST Client or Postman: Send a GET request to http://localhost:8080/hello.

Configuring the Server with application.properties

The file src/main/resources/application.properties is where all Spring Boot configuration lives. Here are the most common settings:

# Change the port (default is 8080)
server.port=8080

# Application name (used in logs and Spring Cloud)
spring.application.name=todo-api

# Show SQL queries in the console (useful during development)
spring.jpa.show-sql=true

# H2 in-memory database (we add more settings in Chapter 35)
spring.datasource.url=jdbc:h2:mem:testdb
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# Enable the H2 web console at /h2-console
spring.h2.console.enabled=true

🔍 How This Code Works

  • server.port=8080 — overrides the default port; change to any free port when 8080 is in use
  • spring.application.name=todo-api — sets the application name shown in logs and used by Spring Cloud service discovery
  • spring.jpa.show-sql=true — prints the SQL Hibernate generates to the console; invaluable for debugging data issues during development
  • spring.datasource.url=jdbc:h2:mem:testdb — connects to an in-memory H2 database named testdb; data resets on every restart
  • spring.datasource.username=sa — H2's default admin username (sa = system administrator)
  • spring.h2.console.enabled=true — exposes a browser-based SQL console at /h2-console for inspecting tables during development
🌍 Real World: Netflix uses Spring Boot for hundreds of microservices. Each service (authentication, recommendations, billing, streaming) is a separate Spring Boot application. Spring Boot's built-in health checks and metrics (/actuator/health) are how Netflix monitors that all 700+ services are running 24/7.
💡 Change the port if 8080 is busy. If another application is already using port 8080, add server.port=9090 (or any free port) to application.properties and restart. No code changes needed.
⚠️ @RestController vs @Controller. @RestController automatically converts your return value to JSON (or plain text) and writes it directly to the HTTP response. @Controller is for server-side rendered HTML views (like Thymeleaf templates). For a REST API, always use @RestController.

Adding a JSON Response

Returning a plain String is fine for testing, but real APIs return JSON. Spring Boot automatically converts a Java object to JSON using the Jackson library (included with spring-boot-starter-web). Just return a Java object from your method:

@GetMapping("/info")
public Map<String, String> info() {
    Map<String, String> response = new HashMap<>();
    response.put("app", "Todo API");
    response.put("version", "1.0");
    return response;   // Spring Boot converts this to: {"app":"Todo API","version":"1.0"}
}

✅ Try It

Add two endpoints to your HelloController:

  1. GET /hello — returns the plain text string "Hello, Spring Boot!"
  2. GET /info — returns a Map with keys "app" and "version"

Run the app with mvn spring-boot:run and test both endpoints in your browser. Then change the port to 9090 in application.properties, restart, and confirm both endpoints work at the new port.

🧠 Multiple Choice Questions

1. What does @SpringBootApplication annotation do?

  • A) Creates the main window
  • B) Combines @Configuration, @EnableAutoConfiguration, and @ComponentScan
  • C) Starts a new thread
  • D) Connects to the database
Show Answer

✅ B) Combines @Configuration, @EnableAutoConfiguration, and @ComponentScan — This single annotation enables Spring Boot's auto-configuration, component scanning, and marks the class as a configuration source.

2. What embedded server does Spring Boot use by default?

  • A) JBoss
  • B) GlassFish
  • C) Tomcat
  • D) Nginx
Show Answer

✅ C) Tomcat — Spring Boot embeds Tomcat by default. No separate server installation needed — the server is bundled inside the JAR.

3. Which annotation marks a class as a Spring REST controller?

  • A) @Service
  • B) @Repository
  • C) @RestController
  • D) @Component
Show Answer

✅ C) @RestController — @RestController = @Controller + @ResponseBody. All methods return data (JSON) directly, not view names.

4. What port does Spring Boot start on by default?

  • A) 3000
  • B) 80
  • C) 443
  • D) 8080
Show Answer

✅ D) 8080 — Spring Boot's embedded Tomcat starts on port 8080 by default. Change it in application.properties: server.port=9090.
