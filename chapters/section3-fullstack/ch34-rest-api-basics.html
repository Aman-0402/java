What is a REST API?

REST stands for Representational State Transfer. A REST API is a set of URL endpoints that a client (browser, React app, mobile app) calls to read or change data on the server.

Think of a restaurant menu. Each item (URL) tells the kitchen (server) what to do. You don't care how the kitchen works — you order and get food (data).

📜 History: REST (Representational State Transfer) was defined by Roy Fielding in his 2000 PhD dissertation at UC Irvine. Fielding was one of the authors of the HTTP specification. He noticed that the web itself already followed certain architectural principles (stateless, resource-based URLs) and formalized them as REST. It became the dominant API style by 2010, replacing SOAP.

Todo Model Class

public class Todo {
    private Long id;
    private String title;
    private boolean completed;

    public Todo() {}

    public Todo(Long id, String title, boolean completed) {
        this.id = id;
        this.title = title;
        this.completed = completed;
    }

    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public boolean isCompleted() { return completed; }
    public void setCompleted(boolean completed) { this.completed = completed; }
}

🔍 How This Code Works

  • private Long id — uses the wrapper type Long (not primitive long) so it can be null before the database assigns an ID
  • public Todo() {} — no-argument constructor required by Jackson so it can deserialize incoming JSON into a Todo object
  • public Todo(Long id, String title, boolean completed) — convenience constructor for creating objects in code without chaining setters
  • Getters and setters follow JavaBean conventions — Jackson uses them to serialize Java fields to JSON keys and back
  • boolean isCompleted() — note the is prefix for boolean getters; Jackson maps this to the JSON key "completed"

Full CRUD Controller

import org.springframework.web.bind.annotation.*;
import java.util.*;

@RestController
@RequestMapping("/api/todos")
@CrossOrigin(origins = "*")
public class TodoController {

    private List<Todo> todos = new ArrayList<>();
    private Long nextId = 1L;

    @GetMapping
    public List<Todo> getAll() { return todos; }

    @GetMapping("/{id}")
    public Todo getOne(@PathVariable Long id) {
        return todos.stream()
            .filter(t -> t.getId().equals(id))
            .findFirst().orElse(null);
    }

    @PostMapping
    public Todo create(@RequestBody Todo todo) {
        todo.setId(nextId++);
        todos.add(todo);
        return todo;
    }

    @PutMapping("/{id}")
    public Todo update(@PathVariable Long id, @RequestBody Todo updated) {
        for (Todo t : todos) {
            if (t.getId().equals(id)) {
                t.setTitle(updated.getTitle());
                t.setCompleted(updated.isCompleted());
                return t;
            }
        }
        return null;
    }

    @DeleteMapping("/{id}")
    public String delete(@PathVariable Long id) {
        todos.removeIf(t -> t.getId().equals(id));
        return "Deleted " + id;
    }
}

🔍 How This Code Works

  • @RestController — every method's return value is automatically serialized to JSON by Jackson and written to the HTTP response body
  • @RequestMapping("/api/todos") — sets the base URL path for all methods in this class, so @GetMapping becomes GET /api/todos
  • @CrossOrigin(origins = "*") — adds CORS headers so browsers on different ports (like React on 3000) can call this API
  • @PathVariable Long id — extracts the {id} segment from the URL (e.g. /api/todos/5 gives id = 5)
  • @RequestBody Todo todo — Jackson deserializes the JSON request body into a Todo object automatically
  • todos.stream().filter(...).findFirst() — searches the in-memory list for a todo matching the given ID
  • todos.removeIf(t -> t.getId().equals(id)) — lambda-based list removal; note .equals() instead of == for object comparison
  • private Long nextId = 1L — manual ID counter since there is no database to auto-increment; replaced in Ch 35 by @GeneratedValue
🔍 Fun Fact: Twitter's REST API receives over 500 million API calls per day. Every tweet, follow, and like is a REST API call. Twitter's API was so influential that it established conventions (JSON responses, resource URLs like /users/:id/tweets) that most modern REST APIs still follow.
💡 @CrossOrigin(origins = "*") lets your React frontend (port 3000) call this API (port 8080). Without it the browser blocks the request with a CORS error.
⚠️ Todos are stored in a List — they vanish on server restart. Chapter 35 adds a real database.

Test with curl

# Create a todo
curl -X POST http://localhost:8080/api/todos \
  -H "Content-Type: application/json" \
  -d '{"title":"Buy milk","completed":false}'

# Get all
curl http://localhost:8080/api/todos

# Update
curl -X PUT http://localhost:8080/api/todos/1 \
  -H "Content-Type: application/json" \
  -d '{"title":"Buy milk","completed":true}'

# Delete
curl -X DELETE http://localhost:8080/api/todos/1

🔍 How This Code Works

  • curl -X POST — specifies the HTTP method; without -X, curl defaults to GET
  • -H "Content-Type: application/json" — tells the server the request body is JSON; Spring Boot requires this header to invoke @RequestBody deserialization
  • -d '{"title":"Buy milk","completed":false}' — the JSON body sent to the server; matches the fields of the Todo class
  • curl http://localhost:8080/api/todos — plain GET request; curl uses GET by default when no -X flag is provided
  • curl -X PUT ... -d '{"completed":true}' — full replacement update; all fields must be provided since PUT replaces the entire resource
  • curl -X DELETE http://localhost:8080/api/todos/1 — the 1 in the URL is extracted by @PathVariable in the controller
🌍 Real World: When you order food on Zomato/Swiggy, at least 10 REST API calls happen: authentication check, restaurant list fetch, menu fetch, cart update, payment processing, order creation, restaurant notification, delivery assignment, location tracking, and notification push. Every tap triggers a REST call to a Spring Boot (or similar) backend.
ℹ️ On Windows use Postman (free GUI) instead of curl — much easier for testing APIs visually.

🧠 Multiple Choice Questions

1. Which HTTP method is used to CREATE a new resource?

  • A) GET
  • B) PUT
  • C) POST
  • D) PATCH
Show Answer

✅ C) POST — POST creates a new resource. The server assigns the ID. GET retrieves, PUT updates (full replace), DELETE removes.

2. What annotation maps a method to HTTP GET requests in Spring Boot?

  • A) @RequestMapping(GET)
  • B) @GetRoute
  • C) @GetMapping
  • D) @HttpGet
Show Answer

✅ C) @GetMapping — Spring shortcut annotations: @GetMapping, @PostMapping, @PutMapping, @DeleteMapping.

3. What does @RequestBody do?

  • A) Reads data from the URL path
  • B) Reads data from query parameters
  • C) Deserializes the HTTP request body (JSON) into a Java object
  • D) Sends data to the client
Show Answer

✅ C) Deserializes the HTTP request body (JSON) into a Java object — Spring Boot uses Jackson to automatically convert incoming JSON to your Java class.

4. What does @PathVariable extract?

  • A) The request body
  • B) Query parameters after ?
  • C) A value from the URL path like /api/todos/{id}
  • D) HTTP headers
Show Answer

✅ C) A value from the URL path like /api/todos/{id} — @GetMapping("/{id}") public Todo get(@PathVariable Long id) extracts the {id} from the URL.
