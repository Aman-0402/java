Chapter 38: Connect React + Spring Boot

Make your React frontend talk to your Spring Boot backend — the bridge that completes your full-stack application.

The Setup

In full-stack development, two servers run at the same time:

  • Spring Boot — runs on http://localhost:8080, serves your REST API
  • React — runs on http://localhost:3000, serves your UI

React fetches data from Spring Boot using JavaScript's built-in fetch() function (or Axios, a popular library).

📜 History: The fetch() API was introduced in 2015 as a modern replacement for XMLHttpRequest (XHR), which dated back to 1999. Microsoft created XHR for Outlook Web Access. Google used XHR for Gmail in 2004, pioneering AJAX (Asynchronous JavaScript and XML). When Gmail launched with live email updates without page reloads, it changed the web forever.

Step 1: Enable CORS in Spring Boot

By default, browsers block requests from localhost:3000 to localhost:8080 because they are different origins. You must tell Spring Boot to allow it:

@RestController
@RequestMapping("/api/todos")
@CrossOrigin(origins = "http://localhost:3000")  // allow React dev server
public class TodoController {
    // ...
}
🔍 Fun Fact: CORS (Cross-Origin Resource Sharing) was introduced in 2006 as a browser security policy. Before CORS, a malicious website could make requests to your bank's API using your session cookies. CORS forces browsers to ask the server "Can this origin access you?" before sending the real request. The browser adds an Origin header; the server responds with Access-Control-Allow-Origin.
💡 For production, replace "http://localhost:3000" with your actual frontend domain, e.g. "https://myapp.netlify.app".

Step 2: Fetch Data in React

Create a React component that loads todos from your Spring Boot API when it mounts:

import { useState, useEffect } from 'react';

function TodoList() {
  const [todos, setTodos] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch('http://localhost:8080/api/todos')
      .then(res => {
        if (!res.ok) throw new Error('Server error: ' + res.status);
        return res.json();
      })
      .then(data => {
        setTodos(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, []); // empty array = run once on mount

  if (loading) return <p>Loading...</p>;
  if (error)   return <p>Error: {error}</p>;

  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id}>
          {todo.completed ? '✅' : '⬜'} {todo.title}
        </li>
      ))}
    </ul>
  );
}

export default TodoList;

🔍 How This Code Works

  • const [error, setError] = useState(null) — dedicated error state; null means no error yet; set to the error message string if the fetch fails
  • fetch('http://localhost:8080/api/todos') — calls the Spring Boot REST API; the browser sends an HTTP GET request to port 8080
  • if (!res.ok) throw new Error('Server error: ' + res.status) — checks that the response was successful (status 200–299) before parsing; prevents silent failures on 404/500 responses
  • .then(data => { setTodos(data); setLoading(false); }) — updates both state variables in sequence; React batches the two setState calls into a single re-render
  • .catch(err => { setError(err.message); setLoading(false); }) — catches both network errors (offline) and thrown errors from the !res.ok check above
  • {todo.completed ? '✅' : '⬜'} — JSX ternary expression; renders different content based on the completed boolean field from the backend JSON
  • export default TodoList — makes the component importable by other files using import TodoList from './TodoList'

Step 3: Create a Todo from React

function AddTodo({ onAdd }) {
  const [title, setTitle] = useState('');

  const handleSubmit = async (e) => {
    e.preventDefault();
    const res = await fetch('http://localhost:8080/api/todos', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ title, completed: false })
    });
    const newTodo = await res.json();
    onAdd(newTodo);
    setTitle('');
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={title}
        onChange={e => setTitle(e.target.value)}
        placeholder="New todo..."
      />
      <button type="submit">Add</button>
    </form>
  );
}

🔍 How This Code Works

  • const handleSubmit = async (e) => { e.preventDefault(); ... } — async enables await inside; e.preventDefault() stops the form from doing a full page reload
  • method: 'POST' — tells fetch to use an HTTP POST request instead of the default GET
  • headers: { 'Content-Type': 'application/json' } — required header so Spring Boot knows to deserialize the body with @RequestBody
  • body: JSON.stringify({ title, completed: false }) — converts the JavaScript object to a JSON string for the request body
  • const newTodo = await res.json() — parses the JSON response; the backend returns the saved entity with its assigned ID
  • onAdd(newTodo) — calls the callback prop to pass the new todo up to the parent component for list update
  • setTitle('') — clears the input field after successful submission
  • onChange={e => setTitle(e.target.value)} — controlled input: every keystroke updates React state, keeping the input in sync with title
⚠️ Always check res.ok before calling res.json(). If the server returns a 404 or 500 error, calling res.json() on an error response can throw confusing errors.

Putting It Together in App.js

import { useState, useEffect } from 'react';

function App() {
  const [todos, setTodos] = useState([]);

  useEffect(() => {
    fetch('http://localhost:8080/api/todos')
      .then(r => r.json())
      .then(setTodos);
  }, []);

  const addTodo = async (title) => {
    const res = await fetch('http://localhost:8080/api/todos', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ title, completed: false })
    });
    const todo = await res.json();
    setTodos(prev => [...prev, todo]);
  };

  return (
    <div>
      <h1>My Todos</h1>
      <input
        onKeyDown={e => e.key === 'Enter' && addTodo(e.target.value)}
        placeholder="Press Enter to add..."
      />
      <ul>
        {todos.map(t => <li key={t.id}>{t.title}</li>)}
      </ul>
    </div>
  );
}

export default App;
🌍 Real World: Every major app you use — Uber, Airbnb, LinkedIn — has a React (or similar) frontend calling a Java/Node/Python backend via API calls exactly like what you just built. The fetch() call you wrote is the same fundamental mechanism powering apps used by billions. The patterns don't change — only the scale does.

🧠 Multiple Choice Questions

1. Which React hook is used to fetch data when a component mounts?

  • A) useState
  • B) useRef
  • C) useEffect
  • D) useFetch
Show Answer

✅ C) useEffect — useEffect(() => { fetch(...) }, []) runs after the component renders. Empty array [] means "run once on mount".

2. What does res.ok check in a fetch response?

  • A) If the response has a body
  • B) If the HTTP status code is in the 200-299 range (success)
  • C) If the response is cached
  • D) If the server is online
Show Answer

✅ B) If the HTTP status code is in the 200-299 range (success) — Always check res.ok before calling res.json(). A 404 or 500 response still returns a response object; res.ok is false.

3. What is CORS?

  • A) A React security module
  • B) Cross-Origin Resource Sharing — a browser policy controlling which domains can call your API
  • C) A type of REST endpoint
  • D) A Java annotation
Show Answer

✅ B) Cross-Origin Resource Sharing — a browser policy controlling which domains can call your API — Without CORS headers, browsers block JavaScript requests to a different origin (domain/port) to prevent cross-site attacks.

4. In Spring Boot, which annotation allows cross-origin requests to a controller?

  • A) @AllowOrigin
  • B) @CORS
  • C) @CrossOrigin
  • D) @EnableCors
Show Answer

✅ C) @CrossOrigin — @CrossOrigin(origins = "http://localhost:3000") adds the necessary Access-Control-Allow-Origin header to responses.
