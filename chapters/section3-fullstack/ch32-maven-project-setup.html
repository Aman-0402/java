Chapter 32: Maven and Project Setup

Learn what Maven is, how to read a pom.xml file, and how to generate a ready-to-run Spring Boot project in under five minutes.

What is Maven?

Before Maven existed, Java developers had to download library JAR files manually, put them in the right folders, and configure everything by hand. Every time a library released an update, developers had to repeat the process. It was painful.

Think of Maven as npm for Java. Just like npm install downloads all the JavaScript packages your project needs, Maven reads a single file called pom.xml and automatically downloads every Java library (called a dependency) your project needs from the internet.

Maven also handles building, testing, and packaging your app — all from the command line.

📜 History: Maven was created by Jason van Zyl in 2002 while working on a project at Apache Software Foundation. Before Maven, Java developers manually downloaded JAR files and managed classpaths. Maven introduced the revolutionary idea of declaring dependencies in XML and downloading them automatically. It was the npm of its era.

Understanding pom.xml

POM stands for Project Object Model. It is an XML file that lives at the root of your project and describes everything Maven needs to know about it.

Here is a minimal Spring Boot pom.xml with explanations:

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
             https://maven.apache.org/xsd/maven-4.0.0.xsd">

  <modelVersion>4.0.0</modelVersion>

  <!-- Who you are -->
  <groupId>com.example</groupId>       <!-- Your organisation/domain (reversed) -->
  <artifactId>todo-api</artifactId>    <!-- Your project name -->
  <version>0.0.1-SNAPSHOT</version>   <!-- Current version -->
  <packaging>jar</packaging>

  <!-- Inherit defaults from Spring Boot -->
  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
    <relativePath/>
  </parent>

  <properties>
    <java.version>17</java.version>
  </properties>

  <!-- Libraries your project needs -->
  <dependencies>

    <!-- Spring Boot Web: lets you build REST APIs -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- Spring Data JPA: database access via Java objects -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>

    <!-- H2: in-memory database, perfect for development -->
    <dependency>
      <groupId>com.h2database</groupId>
      <artifactId>h2</artifactId>
      <scope>runtime</scope>
    </dependency>

    <!-- Spring Boot Test: JUnit + Mockito -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-test</artifactId>
      <scope>test</scope>
    </dependency>

  </dependencies>

  <build>
    <plugins>
      <plugin>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-maven-plugin</artifactId>
      </plugin>
    </plugins>
  </build>

</project>
ℹ️ The <parent> section is key. By inheriting from spring-boot-starter-parent, you get sensible defaults for Java version, plugin configuration, and dependency version management — without specifying version numbers for most Spring libraries yourself.

Generating a Project with Spring Initializr

The fastest way to create a Spring Boot project is through start.spring.io — Spring's official project generator. Follow these steps:

  1. Go to https://start.spring.io in your browser.
  2. Set Project to Maven.
  3. Set Language to Java.
  4. Set Spring Boot version to the latest stable (e.g., 3.2.x).
  5. Fill in Group: com.example, Artifact: todo-api.
  6. Set Java version to 17.
  7. Click ADD DEPENDENCIES and add: Spring Web, Spring Data JPA, H2 Database.
  8. Click GENERATE — a zip file downloads.
  9. Unzip it and open the folder in IntelliJ IDEA or VS Code.
💡 IntelliJ shortcut: In IntelliJ IDEA, go to File → New → Project → Spring Initializr to generate the project directly inside the IDE without visiting the website.

Project Folder Structure

After generating the project you will see this structure:

todo-api/
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/example/todoapi/
│   │   │       └── TodoApiApplication.java   # Main class — entry point
│   │   └── resources/
│   │       ├── application.properties        # Config (port, database URL, etc.)
│   │       ├── static/                       # Static files (HTML, CSS, JS)
│   │       └── templates/                    # Server-side templates (if using Thymeleaf)
│   └── test/
│       └── java/
│           └── com/example/todoapi/
│               └── TodoApiApplicationTests.java
├── pom.xml                                   # Maven config
└── mvnw / mvnw.cmd                           # Maven wrapper (no Maven install needed)

The most important locations:

  • src/main/java — all your Java source code lives here.
  • src/main/resources/application.properties — configuration (server port, database settings).
  • src/test/java — unit and integration tests.
  • pom.xml — dependency and build configuration.
🔍 Fun Fact: Maven Central Repository (repo1.maven.org) hosts over 9 million JAR files (artifacts) from 500,000+ projects. When you add a dependency to pom.xml, Maven downloads it from this repository. It handles over 100 billion downloads per year — one of the most trafficked servers in the world.

Key Maven Commands

Run these in the terminal from your project root directory:

# Compile all Java source files (checks for errors)
mvn compile

# Run all tests
mvn test

# Compile + test + package into a runnable JAR
mvn package

# Run the Spring Boot app directly (most common during development)
mvn spring-boot:run

# Clean all compiled output (fixes weird build caching issues)
mvn clean

# Clean + package (safe full rebuild)
mvn clean package

# If Maven is not installed globally, use the wrapper bundled in the project
./mvnw spring-boot:run        # Linux/Mac
mvnw.cmd spring-boot:run      # Windows
🌍 Real World: Every major Java framework (Spring, Hibernate, Apache Kafka, Elasticsearch) distributes through Maven Central. When Netflix writes new Java microservices, they use Maven/Gradle to pull in hundreds of dependencies. Without Maven, managing those dependencies manually would take days per project.
⚠️ Maven downloads dependencies on first run. The first time you run mvn compile or mvn spring-boot:run, Maven will download all dependency JARs from the internet into a local cache at ~/.m2/repository. This can take a minute or two. Subsequent runs are instant because everything is cached.
ℹ️ What is SNAPSHOT? The version 0.0.1-SNAPSHOT means "work in progress". A SNAPSHOT version is not stable and can change. When you release, you change it to 1.0.0 (no SNAPSHOT suffix).

✅ Try It

Go to https://start.spring.io, generate a project with Spring Web, Spring Data JPA, and H2 Database dependencies. Download and unzip it. Open a terminal inside the project folder and run:

mvn spring-boot:run

Wait for the line Started TodoApiApplication in X.XXX seconds to appear. Then open your browser to http://localhost:8080. You should see a "Whitelabel Error Page" — this means the server is running but has no routes yet. That is expected. You just ran a Spring Boot app! Stop it with Ctrl+C.

🧠 Multiple Choice Questions

1. What is the purpose of pom.xml in a Maven project?

  • A) It's the main Java file
  • B) It defines project metadata, dependencies, and build configuration
  • C) It configures the database
  • D) It's the application startup file
Show Answer

✅ B) It defines project metadata, dependencies, and build configuration — pom.xml (Project Object Model) is Maven's configuration file. It manages all your project's dependencies and build lifecycle.

2. What command runs a Spring Boot application using Maven?

  • A) maven run
  • B) mvn start
  • C) mvn spring-boot:run
  • D) java -run
Show Answer

✅ C) mvn spring-boot:run — This command compiles and starts the Spring Boot application using the embedded Tomcat server.

3. Where does Maven download dependencies from by default?

  • A) Your hard drive
  • B) GitHub
  • C) Maven Central Repository
  • D) Oracle's website
Show Answer

✅ C) Maven Central Repository — Maven downloads JARs from repo.maven.apache.org and caches them in ~/.m2/repository on your machine.

4. What does mvn package do?

  • A) Installs Maven
  • B) Downloads dependencies only
  • C) Compiles code and packages it into a JAR/WAR file
  • D) Deploys the application
Show Answer

✅ C) Compiles code and packages it into a JAR/WAR file — mvn package runs compile, test, and package phases. The output JAR is in the target/ directory.
