Chapter 35: Database + JPA / Hibernate

Connect your Spring Boot API to a real database using JPA so data persists across server restarts.

What is JPA?

JPA (Java Persistence API) is a standard that lets you save Java objects directly to a database without writing SQL. Instead of writing INSERT INTO todos ..., you call todoRepository.save(todo) and JPA handles the SQL for you.

Hibernate is the most popular implementation of JPA — Spring Boot uses it automatically under the hood.

ℹ️ ORM = Object-Relational Mapping. JPA maps Java classes (objects) to database tables (relations). One class = one table. One field = one column.

Step 1: Add Dependencies to pom.xml

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<!-- H2 in-memory database (great for learning) -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>

Step 2: Configure application.properties

# src/main/resources/application.properties

spring.datasource.url=jdbc:h2:mem:tododb
spring.datasource.driver-class-name=org.h2.Driver
spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
spring.h2.console.enabled=true
spring.jpa.show-sql=true
💡 spring.h2.console.enabled=true gives you a web UI at http://localhost:8080/h2-console to inspect your database tables while developing.

Step 3: Create the Entity

Add JPA annotations to your Todo class to tell Hibernate how to map it to a database table:

import jakarta.persistence.*;

@Entity
@Table(name = "todos")
public class Todo {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String title;

    private boolean completed;

    public Todo() {}

    // getters and setters (same as before)
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public boolean isCompleted() { return completed; }
    public void setCompleted(boolean completed) { this.completed = completed; }
}

Step 4: Create the Repository

Create an interface that extends JpaRepository. You get 15+ free CRUD methods without writing a single line of SQL:

import org.springframework.data.jpa.repository.JpaRepository;

public interface TodoRepository extends JpaRepository<Todo, Long> {
    // findAll(), findById(), save(), deleteById() — all free!
}

Step 5: Update the Controller

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/api/todos")
@CrossOrigin(origins = "*")
public class TodoController {

    @Autowired
    private TodoRepository todoRepository;

    @GetMapping
    public List<Todo> getAll() {
        return todoRepository.findAll();
    }

    @PostMapping
    public Todo create(@RequestBody Todo todo) {
        return todoRepository.save(todo);
    }

    @PutMapping("/{id}")
    public Todo update(@PathVariable Long id, @RequestBody Todo updated) {
        updated.setId(id);
        return todoRepository.save(updated);
    }

    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) {
        todoRepository.deleteById(id);
    }
}
⚠️ H2 is an in-memory database — data still resets on restart. For production use MySQL or PostgreSQL. Just change the application.properties datasource URL and add the MySQL driver dependency.

What JPA Annotations Mean

  • @Entity — marks this class as a database table
  • @Table(name = "todos") — sets the table name
  • @Id — marks the primary key field
  • @GeneratedValue(strategy = GenerationType.IDENTITY) — auto-increments the id (1, 2, 3...)
  • @Column(nullable = false) — adds a NOT NULL constraint to the column

✅ Try It

Add the H2 dependency, annotate your Todo class, create the repository, update your controller. Run the app, create some todos via Postman, then visit http://localhost:8080/h2-console and run SELECT * FROM TODOS to see the data in the database.
