Chapter 35: Database + JPA / Hibernate

Connect your Spring Boot API to a real database using JPA so data persists across server restarts.

What is JPA?

JPA (Java Persistence API) is a standard that lets you save Java objects directly to a database without writing SQL. Instead of writing INSERT INTO todos ..., you call todoRepository.save(todo) and JPA handles the SQL for you.

Hibernate is the most popular implementation of JPA — Spring Boot uses it automatically under the hood.

📜 History: Hibernate was created by Gavin King in 2001 as a personal project. He was frustrated with writing repetitive JDBC SQL code. He released it as open source, and it was adopted so widely that Sun Microsystems based the JPA (Java Persistence API) standard on Hibernate's design in 2006. Gavin King was hired by JBoss (later Red Hat) specifically to continue developing Hibernate.
ℹ️ ORM = Object-Relational Mapping. JPA maps Java classes (objects) to database tables (relations). One class = one table. One field = one column.

Step 1: Add Dependencies to pom.xml

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<!-- H2 in-memory database (great for learning) -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>

Step 2: Configure application.properties

# src/main/resources/application.properties

spring.datasource.url=jdbc:h2:mem:tododb
spring.datasource.driver-class-name=org.h2.Driver
spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
spring.h2.console.enabled=true
spring.jpa.show-sql=true

🔍 How This Code Works

  • spring.datasource.url=jdbc:h2:mem:tododb — connects to an in-memory H2 database named tododb; mem: means it lives only in RAM and resets on restart
  • spring.datasource.driver-class-name=org.h2.Driver — tells Spring which JDBC driver to load; swap to com.mysql.cj.jdbc.Driver for MySQL
  • spring.jpa.database-platform=org.hibernate.dialect.H2Dialect — tells Hibernate which SQL dialect to generate; different databases use slightly different SQL syntax
  • spring.h2.console.enabled=true — exposes a web-based SQL browser at /h2-console so you can inspect tables without an external tool
  • spring.jpa.show-sql=true — prints every SQL statement Hibernate executes to the console; essential for debugging unexpected queries
🔍 Fun Fact: Hibernate generates SQL at runtime, which means you can switch from MySQL to PostgreSQL to Oracle by changing one line in application.properties. This database independence is hugely valuable for enterprises. 87% of Java enterprise applications use JPA/Hibernate for database access.
💡 spring.h2.console.enabled=true gives you a web UI at http://localhost:8080/h2-console to inspect your database tables while developing.

Step 3: Create the Entity

Add JPA annotations to your Todo class to tell Hibernate how to map it to a database table:

import jakarta.persistence.*;

@Entity
@Table(name = "todos")
public class Todo {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String title;

    private boolean completed;

    public Todo() {}

    // getters and setters (same as before)
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public boolean isCompleted() { return completed; }
    public void setCompleted(boolean completed) { this.completed = completed; }
}

🔍 How This Code Works

  • @Entity — tells Hibernate this class maps to a database table; Hibernate creates the table on startup if it doesn't exist
  • @Table(name = "todos") — explicitly sets the table name; without it Hibernate defaults to the class name (TODO)
  • @Id — marks this field as the primary key column in the database table
  • @GeneratedValue(strategy = GenerationType.IDENTITY) — lets the database auto-increment the ID (1, 2, 3…) on each insert; you never set id manually
  • @Column(nullable = false) — adds a NOT NULL constraint to the title column; saving a todo without a title throws a database error
  • public Todo() {} — JPA requires a no-argument constructor so Hibernate can instantiate entities when loading rows from the database

Step 4: Create the Repository

Create an interface that extends JpaRepository. You get 15+ free CRUD methods without writing a single line of SQL:

import org.springframework.data.jpa.repository.JpaRepository;

public interface TodoRepository extends JpaRepository<Todo, Long> {
    // findAll(), findById(), save(), deleteById() — all free!
}

🔍 How This Code Works

  • extends JpaRepository<Todo, Long> — first type parameter is the entity class, second is the primary key type; Spring generates the implementation at runtime
  • No method bodies are needed — Spring Data JPA creates a proxy class that implements all CRUD operations automatically
  • findAll() — generates and runs SELECT * FROM todos
  • findById(id) — returns an Optional<Todo>; empty if the row doesn't exist
  • save(todo) — runs INSERT for new entities and UPDATE for existing ones (detected via the ID field)
  • deleteById(id) — runs DELETE FROM todos WHERE id = ?

Step 5: Update the Controller

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/api/todos")
@CrossOrigin(origins = "*")
public class TodoController {

    @Autowired
    private TodoRepository todoRepository;

    @GetMapping
    public List<Todo> getAll() {
        return todoRepository.findAll();
    }

    @PostMapping
    public Todo create(@RequestBody Todo todo) {
        return todoRepository.save(todo);
    }

    @PutMapping("/{id}")
    public Todo update(@PathVariable Long id, @RequestBody Todo updated) {
        updated.setId(id);
        return todoRepository.save(updated);
    }

    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) {
        todoRepository.deleteById(id);
    }
}

🔍 How This Code Works

  • @Autowired private TodoRepository todoRepository — Spring injects a ready-to-use repository instance; no new TodoRepository() needed
  • todoRepository.findAll() — replaces the manual List and generates a SELECT * FROM todos query
  • todoRepository.save(todo) on POST — inserts a new row; Hibernate sets the auto-incremented id on the returned object
  • updated.setId(id) on PUT — sets the ID from the URL path before calling save(); Hibernate detects the existing ID and issues an UPDATE instead of INSERT
  • todoRepository.deleteById(id) — issues DELETE FROM todos WHERE id = ?; throws EmptyResultDataAccessException if the ID doesn't exist
  • public void delete() — returns nothing (HTTP 200 with empty body); a better practice is to return ResponseEntity<Void> with a 204 No Content status
🌍 Real World: eBay's product catalog — 1.5 billion listings — is stored in a database accessed through JPA. Each product is an @Entity class with @Column fields. When you search eBay, Hibernate translates your search into SQL, retrieves entities, and returns them as Java objects. The same ORM pattern you learned in this chapter powers one of the world's largest databases.
⚠️ H2 is an in-memory database — data still resets on restart. For production use MySQL or PostgreSQL. Just change the application.properties datasource URL and add the MySQL driver dependency.

What JPA Annotations Mean

  • @Entity — marks this class as a database table
  • @Table(name = "todos") — sets the table name
  • @Id — marks the primary key field
  • @GeneratedValue(strategy = GenerationType.IDENTITY) — auto-increments the id (1, 2, 3...)
  • @Column(nullable = false) — adds a NOT NULL constraint to the column

🧠 Multiple Choice Questions

1. What does @Entity annotation do?

  • A) Marks a class as a controller
  • B) Marks a Java class as a database table (JPA entity)
  • C) Makes all fields private
  • D) Enables database transactions
Show Answer

✅ B) Marks a Java class as a database table (JPA entity) — Hibernate reads @Entity and creates/maps a corresponding database table for the class.

2. What does JpaRepository<Todo, Long> provide?

  • A) A database connection
  • B) SQL queries you must implement
  • C) 15+ ready-made CRUD methods including findAll, findById, save, deleteById
  • D) A REST controller
Show Answer

✅ C) 15+ ready-made CRUD methods including findAll, findById, save, deleteById — Spring Data JPA generates implementations for all these methods automatically at runtime.

3. What does @GeneratedValue(strategy = GenerationType.IDENTITY) do?

  • A) Creates a random UUID
  • B) Lets you set the ID manually
  • C) Auto-increments the primary key (1, 2, 3...)
  • D) Copies the ID from another table
Show Answer

✅ C) Auto-increments the primary key (1, 2, 3...) — IDENTITY delegates ID generation to the database (using AUTO_INCREMENT in MySQL or SERIAL in PostgreSQL).

4. What is ORM?

  • A) Object Resource Management
  • B) A security protocol
  • C) Object-Relational Mapping — maps Java objects to database tables automatically
  • D) A type of SQL query
Show Answer

✅ C) Object-Relational Mapping — maps Java objects to database tables automatically — ORM eliminates the need to write most SQL manually, mapping @Entity classes to tables and fields to columns.
