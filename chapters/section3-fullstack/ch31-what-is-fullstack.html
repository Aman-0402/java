📙 Section 3 — Java Full Stack

Chapter 31: What is Full Stack Development?

Understand the three layers of every web application — frontend, backend, and database — and how they work together to serve a user's request.

The Restaurant Analogy

Imagine you walk into a restaurant and order a burger. Here is what happens:

  1. You (the customer) read the menu and place an order with the waiter.
  2. The waiter carries your order to the kitchen and brings back the burger.
  3. The kitchen looks up the recipe and prepares the burger from ingredients in the fridge.

A web application works exactly the same way:

  • Customer = Browser (Frontend) — what the user sees and interacts with: buttons, forms, pages.
  • Waiter = Backend Server — receives requests, applies business logic, and sends back responses.
  • Kitchen + Fridge = Database — stores all the data (users, products, orders) and retrieves it on demand.

The Three Layers Explained

Frontend — What the User Sees

The frontend is the part of the application that runs in the user's browser. It is built with HTML (structure), CSS (style), and JavaScript (interactivity). In Section 3 we will use React — a JavaScript library that makes building interactive UIs much faster.

The frontend never talks directly to the database. It only talks to the backend by sending HTTP requests.

Backend — The Server Logic

The backend is a program running on a server. Its job is to:

  • Receive requests from the frontend (e.g., "give me all todos")
  • Apply business rules (e.g., "only show todos belonging to this user")
  • Fetch or save data to the database
  • Send back a response (usually JSON)

In Section 3 we will use Spring Boot — a Java framework that makes building backends fast and straightforward.

Database — Persistent Storage

The database stores data permanently. When a user registers, their email and password are saved to the database. When they log in next week, the backend reads that data back. Without a database, all data would vanish the moment the server restarts.

We will use H2 (an in-memory database, perfect for learning) and later you can swap it for MySQL or PostgreSQL with a single config change.

📜 History: The term "full-stack developer" was coined around 2008 at Facebook. Facebook engineers were expected to work across the entire stack — from PHP backend to JavaScript frontend to MySQL databases. The term spread industrywide after a 2012 blog post by Facebook engineer Carlos Bueno titled "The Full Stack."

The Request-Response Cycle

Every web interaction follows this cycle:

  1. User clicks a button in the browser — the frontend sends an HTTP request to the backend.
  2. Backend receives the request, processes it, and queries the database if needed.
  3. Database returns the data to the backend.
  4. Backend formats the data as JSON and sends it back to the frontend.
  5. Frontend displays the data to the user.
🔍 Fun Fact: A typical web request touches at least 5 layers: DNS server, load balancer, web server, application server, database. A "full stack" developer understands all of these. At large companies (Google, Amazon), specialists handle each layer. At startups, one developer handles all of them.
ℹ️ JSON (JavaScript Object Notation) is the universal language that backends and frontends use to exchange data. Example: {"name": "Alice", "age": 25}. It is readable by both Java and JavaScript — that is why it became the standard.

Database (prod) MySQL / PostgreSQL Production-grade persistent storage

Frontend React 18 Interactive user interface in the browser
API Format JSON over HTTP Communication between frontend and backend
🌍 Real World: Instagram was built by just 13 engineers when it was acquired by Facebook for $1 billion in 2012. All 13 were full-stack developers. The backend was Python/Django, frontend was HTML/CSS/JS, database was PostgreSQL. 13 people built an app used by 30 million users. Full-stack skills made this possible.
💡 Full stack does not mean you do everything alone. In a real company, frontend and backend teams are often separate. But knowing both sides makes you far more valuable — you can debug issues that cross the boundary, and you can build complete features yourself.
⚠️ Do not skip the fundamentals. Spring Boot hides a lot of complexity. If you jump straight into frameworks without understanding HTTP requests, JSON, and basic server concepts, you will be lost when things go wrong. Read this chapter slowly — everything in Section 3 builds on these ideas.

Prerequisites Checklist

Before continuing, make sure you have:

  • JDK 17 or higher installed — run java -version in the terminal to confirm.
  • Maven installed — run mvn -version (or use the Maven wrapper bundled with your project).
  • An IDE — IntelliJ IDEA Community Edition (recommended) or VS Code with the Java Extension Pack.
  • A terminal you are comfortable using (Command Prompt, PowerShell, or the one built into your IDE).

✅ Try It

Open your browser and visit any website (e.g., github.com). Open the browser developer tools with F12, go to the Network tab, and refresh the page. You will see a list of HTTP requests the browser made. Click on a request and look at its Response tab — if it shows JSON data you are looking at a real backend API call. Find at least one JSON response and identify what kind of data it contains.

🧠 Multiple Choice Questions

1. Which layer handles business logic and database operations in a full-stack app?

  • A) Frontend (React)
  • B) Backend (Spring Boot)
  • C) CSS
  • D) The browser
Show Answer

✅ B) Backend (Spring Boot) — The backend processes requests, applies business rules, and communicates with the database. The frontend only handles UI.

2. What protocol do frontend and backend typically use to communicate?

  • A) FTP
  • B) SMTP
  • C) HTTP/HTTPS
  • D) SSH
Show Answer

✅ C) HTTP/HTTPS — REST APIs communicate over HTTP. The frontend sends HTTP requests (GET, POST, etc.) and the backend responds with JSON.

3. What does "full stack" mean?

  • A) A technology that includes everything in one file
  • B) A developer or architecture that covers both frontend (UI) and backend (server/database)
  • C) A specific framework
  • D) A type of database
Show Answer

✅ B) A developer or architecture that covers both frontend (UI) and backend (server/database) — Full stack = both sides of the client-server model.

4. In a typical Java full-stack app, what role does JSON play?

  • A) It styles the UI
  • B) It defines the database schema
  • C) It's the data format used to send/receive data between frontend and backend
  • D) It's a Java testing library
Show Answer

✅ C) It's the data format used to send/receive data between frontend and backend — Spring Boot automatically converts Java objects to JSON and back using Jackson library.
