Chapter 30: Graphs, BFS, and DFS

Graphs model anything with connections — cities and roads, social networks, web pages and links — and BFS and DFS are the two fundamental algorithms for exploring them.

📜 History: Graph theory was invented by Leonhard Euler in 1736 to solve the "Seven Bridges of Königsberg" problem — could you walk through a city crossing each bridge exactly once? Euler proved it was impossible and in doing so invented graph theory. 288 years later, every GPS, social network, and internet router uses his mathematical framework.

What Is a Graph?

Imagine a map of cities connected by roads. Each city is a point, and each road is a line between two points. That is a graph. More formally:

  • A vertex (also called a node) is a point in the graph — a city, a person, a web page.
  • An edge is a connection between two vertices — a road, a friendship, a hyperlink.

Graphs are the most general data structure you have studied so far. Trees are actually a special case of a graph — a tree is a connected graph with no cycles.

🔍 Fun Fact: Facebook's social graph has 3 billion nodes (users) and hundreds of billions of edges (friendships). The average distance between any two Facebook users is 3.57 degrees of separation — meaning you're connected to anyone on Earth through fewer than 4 mutual friends. Facebook computed this using distributed BFS across hundreds of servers.

Representing a Graph in Java: Adjacency List

The most practical representation for most graph problems is an adjacency list: a map where each vertex maps to a list of its neighbours. This is memory-efficient for sparse graphs (few edges) and fast for iterating over neighbours. 

import java.util.*;

public class Graph {

    // Map: vertex → list of adjacent vertices
    private Map<Integer, List<Integer>> adjacencyList;

    public Graph() {
        adjacencyList = new HashMap<>();
    }

    // Add a vertex (creates an empty neighbour list if needed)
    public void addVertex(int vertex) {
        adjacencyList.putIfAbsent(vertex, new ArrayList<>());
    }

    // Add an undirected edge between u and v
    public void addEdge(int u, int v) {
        adjacencyList.putIfAbsent(u, new ArrayList<>());
        adjacencyList.putIfAbsent(v, new ArrayList<>());
        adjacencyList.get(u).add(v);
        adjacencyList.get(v).add(u);  // remove this line for a directed graph
    }

    public List<Integer> getNeighbors(int vertex) {
        return adjacencyList.getOrDefault(vertex, new ArrayList<>());
    }

    public Set<Integer> getVertices() {
        return adjacencyList.keySet();
    }

    // Build the example graph from the ASCII diagram above
    public static Graph buildExampleGraph() {
        Graph g = new Graph();
        g.addEdge(1, 2);
        g.addEdge(1, 3);
        g.addEdge(2, 4);
        g.addEdge(2, 5);
        g.addEdge(3, 4);
        g.addEdge(3, 6);
        g.addEdge(4, 5);
        return g;
    }
}

BFS — Breadth-First Search

BFS explores the graph level by level, starting from a source vertex. It visits all neighbours of the starting vertex first, then all neighbours of those neighbours, and so on — like ripples spreading outward from a stone dropped in water.

BFS uses a Queue (FIFO). It also needs a visited set to avoid revisiting vertices and getting stuck in cycles.

BFS finds the shortest path (in terms of number of edges) from the source to any reachable vertex in an unweighted graph. That is why it powers "degrees of separation" in social networks and shortest-route features.

BFS Step Trace — Starting from Vertex 1

Graph:  1-2, 1-3, 2-4, 2-5, 3-4, 3-6, 4-5

Start: source=1, visited={1}, queue=[1]

Step 1: Dequeue 1. Visit 1.
        Neighbours: 2, 3 (not visited)
        Enqueue 2, 3 → queue=[2, 3], visited={1,2,3}

Step 2: Dequeue 2. Visit 2.
        Neighbours: 1(visited), 4, 5 (not visited)
        Enqueue 4, 5 → queue=[3, 4, 5], visited={1,2,3,4,5}

Step 3: Dequeue 3. Visit 3.
        Neighbours: 1(visited), 4(visited), 6 (not visited)
        Enqueue 6 → queue=[4, 5, 6], visited={1,2,3,4,5,6}

Step 4: Dequeue 4. Visit 4.
        Neighbours: 2,3,5 (all visited)
        queue=[5, 6]

Step 5: Dequeue 5. Visit 5.
        Neighbours: 2,4 (all visited)
        queue=[6]

Step 6: Dequeue 6. Visit 6.
        Neighbours: 3 (visited)
        queue=[]

BFS order: 1 → 2 → 3 → 4 → 5 → 6
Level 0: 1
Level 1: 2, 3       (1 edge from source)
Level 2: 4, 5, 6    (2 edges from source)

BFS Java Implementation

import java.util.*;

public class BFS {

    public static void bfs(Graph graph, int source) {
        Set<Integer> visited = new HashSet<>();
        Queue<Integer> queue = new LinkedList<>();

        // Start from the source vertex
        visited.add(source);
        queue.add(source);

        System.out.print("BFS order: ");

        while (!queue.isEmpty()) {
            int current = queue.poll();  // dequeue the front element
            System.out.print(current + " ");

            // Enqueue all unvisited neighbours
            for (int neighbor : graph.getNeighbors(current)) {
                if (!visited.contains(neighbor)) {
                    visited.add(neighbor);
                    queue.add(neighbor);
                }
            }
        }
        System.out.println();
    }

    public static void main(String[] args) {
        Graph g = Graph.buildExampleGraph();
        bfs(g, 1);  // BFS order: 1 2 3 4 5 6
    }
}

DFS — Depth-First Search

DFS explores the graph by going as deep as possible down one path before backtracking to try another. Think of navigating a maze: you follow one corridor until you hit a dead end, then backtrack to the last junction and try a different corridor.

The recursive version of DFS uses the call stack implicitly, making the code very clean. You also need a visited set for the same reason as BFS.

DFS is useful for: detecting cycles, topological sorting, finding connected components, solving mazes, and exploring all possible paths.

DFS Step Trace — Starting from Vertex 1

Graph:  1-2, 1-3, 2-4, 2-5, 3-4, 3-6, 4-5

dfs(1):
  Visit 1. Explore neighbours: [2, 3]
  → dfs(2):
      Visit 2. Explore neighbours: [1(visited), 4, 5]
      → dfs(4):
          Visit 4. Explore neighbours: [2(visited), 3, 5]
          → dfs(3):
              Visit 3. Explore neighbours: [1(visited), 4(visited), 6]
              → dfs(6):
                  Visit 6. Neighbours: [3(visited)] → return
              return
          → dfs(5):
              Visit 5. Neighbours: [2(visited), 4(visited)] → return
          return
      → dfs(5): already visited, skip
      return
  → dfs(3): already visited, skip

DFS order: 1 → 2 → 4 → 3 → 6 → 5

DFS Java Implementation (Recursive)

import java.util.*;

public class DFS {

    public static void dfs(Graph graph, int current, Set<Integer> visited) {
        visited.add(current);
        System.out.print(current + " ");

        // Recursively explore each unvisited neighbour
        for (int neighbor : graph.getNeighbors(current)) {
            if (!visited.contains(neighbor)) {
                dfs(graph, neighbor, visited);
            }
        }
    }

    public static void main(String[] args) {
        Graph g = Graph.buildExampleGraph();

        Set<Integer> visited = new HashSet<>();
        System.out.print("DFS order: ");
        dfs(g, 1, visited);  // DFS order: 1 2 4 3 6 5
        System.out.println();
    }
}

BFS vs DFS — When to Use Which

PropertyBFSDFS
Data structureQueue (FIFO)Stack (call stack / explicit)
ExploresLevel by levelDeep first, then backtrack
Shortest path?Yes (unweighted)No
Use whenShortest path, nearest neighbourAll paths, cycle detection, topological sort
MemoryMore (stores entire level)Less (stores one path at a time)
Time complexityO(V + E)O(V + E)

Both BFS and DFS have the same time complexity — O(V + E) where V is the number of vertices and E is the number of edges — because each vertex and each edge is visited exactly once.

🌍 Real World: Google Maps uses Dijkstra's algorithm (a graph shortest-path algorithm) to find the fastest route. Your city is a graph: intersections = vertices, roads = edges with weights (distance/time). When you ask for directions, Google searches this graph — typically across hundreds of thousands of nodes — and finds the optimal path in under a second.
💡 Tip — Always Track Visited Vertices: In a graph with cycles (which is most real-world graphs), forgetting the visited set will cause infinite loops. Unlike a tree, a graph can have multiple paths to the same node — you must remember where you have already been.
⚠️ Common Mistake — Disconnected Graphs: If the graph is not fully connected, starting BFS or DFS from a single vertex will only visit the vertices reachable from that vertex. To ensure every vertex is visited in a disconnected graph, wrap your BFS/DFS in a loop over all vertices and only start a traversal from unvisited ones. This pattern is used to count connected components.

Putting It All Together — Full Example

import java.util.*;

public class GraphDemo {

    static Map<Integer, List<Integer>> graph = new HashMap<>();

    static void addEdge(int u, int v) {
        graph.computeIfAbsent(u, k -> new ArrayList<>()).add(v);
        graph.computeIfAbsent(v, k -> new ArrayList<>()).add(u);
    }

    // BFS — finds shortest path (hop count) from source to all vertices
    static void bfs(int source) {
        Set<Integer> visited = new HashSet<>();
        Queue<Integer> queue = new LinkedList<>();
        visited.add(source);
        queue.add(source);
        System.out.print("BFS: ");
        while (!queue.isEmpty()) {
            int node = queue.poll();
            System.out.print(node + " ");
            for (int nb : graph.getOrDefault(node, List.of())) {
                if (!visited.contains(nb)) {
                    visited.add(nb);
                    queue.add(nb);
                }
            }
        }
        System.out.println();
    }

    // DFS — recursive
    static void dfs(int node, Set<Integer> visited) {
        visited.add(node);
        System.out.print(node + " ");
        for (int nb : graph.getOrDefault(node, List.of())) {
            if (!visited.contains(nb)) {
                dfs(nb, visited);
            }
        }
    }

    public static void main(String[] args) {
        addEdge(1, 2); addEdge(1, 3); addEdge(2, 4);
        addEdge(2, 5); addEdge(3, 4); addEdge(3, 6); addEdge(4, 5);

        bfs(1);  // BFS: 1 2 3 4 5 6

        System.out.print("DFS: ");
        dfs(1, new HashSet<>());  // DFS: 1 2 4 3 6 5
        System.out.println();
    }
}
