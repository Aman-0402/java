Chapter 28: Trees and Binary Search Trees

A tree is a hierarchical data structure where every node has at most one parent — understanding trees unlocks file systems, databases, compilers, and much more.

📜 History: Tree data structures were first formally described by Arthur Cayley in 1857 in mathematics. In computing, they appeared in 1960 with LISP's S-expressions (which are essentially trees). Binary Search Trees were analyzed by Hibbard in 1962. Today, trees are in virtually every piece of software you use.

What Is a Tree?

Think of a family tree. There is one person at the top — a great-grandparent — and they have children, who have children of their own, and so on. No child has two parents in the tree (it is not a social graph, it is a strict hierarchy). That is exactly how a tree data structure works.

Unlike linked lists, stacks, and queues — which are all linear (one element follows another in a line) — a tree is hierarchical: each element can branch into multiple children.

Terminology

TermMeaning
NodeA single element in the tree, holding data.
RootThe topmost node; the tree's entry point. Has no parent.
LeafA node with no children (at the bottom of the tree).
ParentA node that has one or more children below it.
ChildA node directly connected below a parent.
SubtreeAny node together with all its descendants.
HeightThe number of edges on the longest path from root to a leaf.
DepthThe number of edges from the root to a particular node.
LevelAll nodes at the same depth. Root is level 0.

Binary Tree

A binary tree is a tree where each node has at most two children, called the left child and the right child. "At most" means a node can have zero, one, or two children — never three or more. 

Binary Tree — general (no ordering rule):

           10          ← root (depth 0)
          /  \
         5    15       ← depth 1
        / \     \
       3   7    20     ← depth 2 (leaves: 3, 7, 20)

Height = 2  (two edges from root to deepest leaf)

Binary Search Tree (BST)

A Binary Search Tree is a binary tree with one extra rule that makes searching fast:

For every node N: all values in the left subtree are less than N.value, and all values in the right subtree are greater than N.value.

This ordering rule means you can find any value using binary-search-like logic: at each node, decide whether to go left or right. You eliminate half the remaining tree at each step, giving O(log n) average-case search.

BST Operations

Insert

Start at the root. If the new value is less than the current node, go left; if greater, go right. Repeat until you reach an empty spot (null), then place the new node there. 

Insert 5 into the BST above:

  Start at 8: 5 < 8 → go left
  At 3:       5 > 3 → go right
  At 6:       5 < 6 → go left
  At 4:       5 > 4 → go right
  4.right is null → insert 5 here

            8
           / \
          3   10
         / \    \
        1   6   14
           / \
          4   7
           \
            5   ← newly inserted

Search

Same logic as insert: compare the target with the current node, go left if smaller, right if greater, return true when you find it, return false when you hit null. 

Search for 6:

  Start at 8: 6 < 8 → go left
  At 3:       6 > 3 → go right
  At 6:       6 == 6 → FOUND ✓  (2 comparisons from root)

Search for 5 (not in tree):

  Start at 8: 5 < 8 → left
  At 3:       5 > 3 → right
  At 6:       5 < 6 → left
  At 4:       5 > 4 → right
  null → NOT FOUND

In-Order Traversal

There are three classic ways to traverse (visit every node in) a binary tree:

  • In-order: left subtree → current node → right subtree
  • Pre-order: current node → left subtree → right subtree
  • Post-order: left subtree → right subtree → current node

The magic of in-order traversal on a BST: it visits nodes in ascending sorted order. This is guaranteed by the BST property — every node in the left subtree is smaller, so you visit smaller values first, then the current node, then larger values. 

In-order traversal of BST above:
  Visit 1, 3, 4, 6, 7, 8, 10, 14

  That is the sorted order of all inserted values. ✓

Java Implementation

public class BinarySearchTree {

    // Each node holds a value and references to left and right children
    static class Node {
        int value;
        Node left;
        Node right;

        Node(int value) {
            this.value = value;
            this.left = null;
            this.right = null;
        }
    }

    private Node root;

    // Insert a value into the BST
    public void insert(int value) {
        root = insertRecursive(root, value);
    }

    private Node insertRecursive(Node current, int value) {
        // Base case: we found the empty spot — create a new node here
        if (current == null) {
            return new Node(value);
        }

        if (value < current.value) {
            current.left = insertRecursive(current.left, value);
        } else if (value > current.value) {
            current.right = insertRecursive(current.right, value);
        }
        // If value == current.value, duplicates are ignored here

        return current;
    }

    // Search for a value — returns true if found, false otherwise
    public boolean search(int value) {
        return searchRecursive(root, value);
    }

    private boolean searchRecursive(Node current, int value) {
        if (current == null) {
            return false;  // reached a dead end — not found
        }
        if (value == current.value) {
            return true;   // found!
        }
        if (value < current.value) {
            return searchRecursive(current.left, value);
        } else {
            return searchRecursive(current.right, value);
        }
    }

    // In-order traversal: visits nodes in sorted ascending order
    public void inOrder() {
        inOrderRecursive(root);
        System.out.println();
    }

    private void inOrderRecursive(Node current) {
        if (current == null) return;

        inOrderRecursive(current.left);   // visit left subtree
        System.out.print(current.value + " ");  // visit current node
        inOrderRecursive(current.right);  // visit right subtree
    }

    public static void main(String[] args) {
        BinarySearchTree bst = new BinarySearchTree();

        // Build the tree
        int[] values = {8, 3, 10, 1, 6, 14, 4, 7};
        for (int v : values) {
            bst.insert(v);
        }

        // In-order traversal prints sorted output
        System.out.print("In-order: ");
        bst.inOrder();  // 1 3 4 6 7 8 10 14

        // Search
        System.out.println("Search 6: " + bst.search(6));   // true
        System.out.println("Search 5: " + bst.search(5));   // false
        System.out.println("Search 14: " + bst.search(14)); // true
    }
}

🔍 How This Code Works

  • root = insertRecursive(root, value) — the public insert reassigns root to whatever the recursive helper returns; this pattern cleanly handles the empty-tree case
  • if (current == null) return new Node(value) — base case for insert; when recursion reaches an empty slot, a new node is created and returned up the call chain to be attached
  • current.left = insertRecursive(current.left, value) — the returned node (new or unchanged) is always re-linked to its parent, keeping the tree intact even when only a distant leaf changes
  • if (current == null) return false in search — hitting a null pointer means the path in the BST that would contain the target is empty; the value is not in the tree
  • inOrderRecursive(current.left) before printing — recursing left first means smaller values are visited before the current node, producing ascending order
  • inOrderRecursive(current.right) after printing — recursing right after the current node ensures larger values appear last
  • In-order traversal of any BST always yields values in sorted ascending order — a consequence of the BST ordering rule
🔍 Fun Fact: The HTML of every web page is stored as a tree called the DOM (Document Object Model). Your browser parses HTML into a tree of nodes (<html> → <body> → <div> → <p>). When JavaScript changes a webpage, it's modifying tree nodes. Java's XML parsers work the same way.
💡 Tip — BST Gives You a Free Sort: If you insert n elements into a BST one at a time and then do an in-order traversal, you get them back in sorted order. This is called Tree Sort and runs in O(n log n) for a balanced tree — the same complexity as Merge Sort.
🌍 Real World: Chess engines (like Stockfish) build a "game tree" where each node is a board position and each edge is a legal move. The engine searches this tree to find the best move. A chess game tree has roughly 10^120 possible positions — more than atoms in the observable universe. Algorithms like Alpha-Beta pruning prune this tree to make it searchable.
⚠️ Common Mistake — Unbalanced BSTs: If you insert elements in sorted order (e.g., 1, 2, 3, 4, 5), every node goes to the right child of the previous one. The BST degenerates into a linked list, and search becomes O(n) — no better than linear search. Real-world implementations use self-balancing trees like AVL trees or Red-Black trees (which Java's TreeMap uses) to keep height at O(log n) automatically.

🧠 Multiple Choice Questions

1. In a Binary Search Tree, where is the smallest value?

  • A) The root
  • B) The rightmost node
  • C) The leftmost node
  • D) Random
Show Answer

✅ C) The leftmost node — In a BST, all left children are smaller. Traversing left repeatedly reaches the minimum.

2. What does inOrder traversal of a BST produce?

  • A) Nodes in insertion order
  • B) Nodes in sorted (ascending) order
  • C) Nodes from deepest to shallowest
  • D) Random order
Show Answer

✅ B) Nodes in sorted (ascending) order — inOrder = left → root → right. For a BST, this produces all values in ascending order.

3. What is the height of a balanced binary tree with 7 nodes?

  • A) 7
  • B) 3
  • C) 2
  • D) 4
Show Answer

✅ B) 3 — A balanced tree with 7 nodes has height 3 (root + 2 more levels). Height = floor(log₂(n)) + 1 = floor(log₂(7)) + 1 = 3.

4. What is the time complexity of search in a balanced BST?

  • A) O(n)
  • B) O(1)
  • C) O(log n)
  • D) O(n log n)
Show Answer

✅ C) O(log n) — In a balanced BST, each comparison eliminates half the remaining tree. Unbalanced BST can degrade to O(n).
