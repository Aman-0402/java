Chapter 25: Queue

A queue is a First-In, First-Out (FIFO) data structure where items are added at the back and removed from the front — just like a ticket counter line.

The Ticket Counter Analogy

Imagine a queue at a railway ticket counter. The first person to arrive is the first person to be served. New arrivals join at the back. Nobody cuts in line (ideally). This is exactly how a queue data structure behaves: first in, first out.

The three core operations are:

  • enqueue (offer) — add an item to the back of the queue
  • dequeue (poll) — remove and return the item at the front
  • peek — look at the front item without removing it

Queue Structure — ASCII Diagram

After enqueue(A), enqueue(B), enqueue(C):

  FRONT                              BACK
   │                                  │
   ▼                                  ▼
 ┌─────┐     ┌─────┐     ┌─────┐
 │  A  │ ──► │  B  │ ──► │  C  │
 └─────┘     └─────┘     └─────┘

poll() → removes A (front), queue becomes [B, C]
poll() → removes B,          queue becomes [C]
offer(D) →                   queue becomes [C, D]
peek()  → returns C (no removal)

Implementing a Queue in Java

Java's LinkedList class implements the Queue interface, making it the standard way to create a queue. Use the Queue interface type for the variable declaration.

import java.util.LinkedList;
import java.util.Queue;

Queue<String> queue = new LinkedList<>();

// offer() — add to back (returns false if queue is capacity-limited)
queue.offer("Alice");
queue.offer("Bob");
queue.offer("Carol");

// peek() — look at front without removing
System.out.println(queue.peek()); // Alice

// poll() — remove from front (returns null if empty)
System.out.println(queue.poll()); // Alice
System.out.println(queue.poll()); // Bob

System.out.println("Remaining: " + queue); // [Carol]
System.out.println("Size: " + queue.size()); // 1
💡 Prefer offer() over add() and poll() over remove(). The offer/poll/peek methods return special values (false or null) on failure, while add/remove throw exceptions — the special-value versions are usually easier to work with in loops.

Use Cases for Queues

  • Print queue — documents print in the order they were submitted.
  • Task scheduling — OS processes CPU tasks in queue order.
  • Breadth-First Search (BFS) — explores a graph level by level using a queue.
  • Customer service simulation — customers wait in line to be served.
  • Message queues — systems like Kafka use FIFO queues for event streaming.

Full Example: Ticket Queue Simulation

Simulate a railway ticket counter serving 5 customers. Each customer arrives with a name and the simulation processes them one at a time.

Step-by-Step Trace

Customers arrive:   [Alice, Bob, Carol, Dave, Eve]  (front → back)

Serve Alice: poll() → "Now serving: Alice"   Queue: [Bob, Carol, Dave, Eve]
Serve Bob:   poll() → "Now serving: Bob"     Queue: [Carol, Dave, Eve]
Serve Carol: poll() → "Now serving: Carol"   Queue: [Dave, Eve]
Serve Dave:  poll() → "Now serving: Dave"    Queue: [Eve]
Serve Eve:   poll() → "Now serving: Eve"     Queue: []

Queue empty — counter closes.

import java.util.LinkedList;
import java.util.Queue;

public class TicketQueue {

    public static void main(String[] args) {
        Queue<String> counter = new LinkedList<>();

        // Customers arrive and join the queue
        String[] arrivals = {"Alice", "Bob", "Carol", "Dave", "Eve"};
        for (String name : arrivals) {
            counter.offer(name);
            System.out.println(name + " joined the queue. Queue size: " + counter.size());
        }

        System.out.println("\n--- Counter opens ---");

        // Serve each customer in order
        while (!counter.isEmpty()) {
            String customer = counter.poll();
            System.out.println("Now serving: " + customer);

            // Peek at who is next (if anyone)
            if (!counter.isEmpty()) {
                System.out.println("  Next up: " + counter.peek());
            }
        }

        System.out.println("\nAll customers served. Counter closes.");
    }
}

Priority Queue — A Brief Introduction

A regular queue serves everyone in arrival order. A priority queue serves the highest-priority item first, regardless of order. Java's PriorityQueue class implements this using a heap internally.

import java.util.PriorityQueue;

// Default: smallest number = highest priority (min-heap)
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.offer(30);
pq.offer(10);
pq.offer(20);

System.out.println(pq.poll()); // 10 — smallest first
System.out.println(pq.poll()); // 20
System.out.println(pq.poll()); // 30
ℹ️ Priority queues are widely used in Dijkstra's shortest path algorithm, hospital triage systems, and CPU task scheduling. They are covered in depth when you reach graph algorithms.
⚠️ PriorityQueue in Java does not guarantee the order of elements with equal priority. If you need stable ordering (ties broken by insertion order), you need a custom Comparator that includes a sequence number.
