Chapter 24: Stack

A stack is a Last-In, First-Out (LIFO) data structure where you can only add or remove items from the top — like a stack of plates.

📜 History: The stack data structure was invented by Friedrich L. Bauer in 1957. He called it a "cellar" and used it to evaluate arithmetic expressions. In 1957, he was working on the ALGOL compiler — the first language to use a stack for function calls. Every modern CPU has a hardware stack built in.

The Plate Stack Analogy

Imagine a stack of plates in a cafeteria. You always place a new plate on top, and you always take from the top. You cannot grab a plate from the middle without removing all the plates above it first. This "last in, first out" behaviour is exactly how a stack data structure works.

The three core operations are:

  • push — add an item to the top
  • pop — remove and return the top item
  • peek — look at the top item without removing it

Stack Structure — ASCII Diagram

After push(10), push(20), push(30):

      TOP
   ┌───────┐
   │  30   │  ← peek() returns 30, pop() removes this
   ├───────┤
   │  20   │
   ├───────┤
   │  10   │  ← bottom (first item pushed)
   └───────┘

pop() → removes 30, stack becomes [10, 20]
pop() → removes 20, stack becomes [10]
pop() → removes 10, stack is empty
pop() on empty stack → throws NoSuchElementException

Implementing a Stack with ArrayDeque

Java has a legacy Stack class but it is outdated and thread-synchronized unnecessarily. The modern, recommended approach is ArrayDeque, which is faster and more flexible.

import java.util.ArrayDeque;
import java.util.Deque;

Deque<Integer> stack = new ArrayDeque<>();

// push — add to top
stack.push(10);
stack.push(20);
stack.push(30);

// peek — look at top without removing
System.out.println(stack.peek()); // 30

// pop — remove from top
System.out.println(stack.pop());  // 30
System.out.println(stack.pop());  // 20

System.out.println("Remaining: " + stack); // [10]
System.out.println("Empty? " + stack.isEmpty()); // false
💡 Always declare the variable as Deque<T> (the interface) rather than ArrayDeque<T> (the implementation). This makes it easy to swap implementations later and signals to readers that only stack operations are intended.
🔍 Fun Fact: Every time you call a method in Java, a new "stack frame" is pushed onto the JVM's call stack. This is why StackOverflowError happens with deep recursion — the stack runs out of space. The default JVM stack size is ~512KB, fitting roughly 5,000–10,000 method calls.

Use Cases for Stacks

  • Undo/Redo — each action is pushed onto a stack; Ctrl+Z pops the last action.
  • Browser back button — each visited page is pushed; clicking Back pops to the previous page.
  • Expression evaluation — compilers use stacks to evaluate arithmetic like 3 + (4 * 2).
  • Call stack — the JVM itself uses a stack to track method calls (as you saw in Chapter 22).
  • Bracket matching — check whether brackets in code are properly paired.

Full Example: Bracket Matching

A common interview problem: given a string like "({[]})", determine whether every opening bracket has a matching closing bracket in the correct order. A stack is the perfect tool.

Step-by-Step Trace on "({[]})"

Input: ( { [ ] } )
       1 2 3 4 5 6

Step 1: '('  → opening → push '('    stack: ['(']
Step 2: '{'  → opening → push '{'    stack: ['(', '{']
Step 3: '['  → opening → push '['    stack: ['(', '{', '[']
Step 4: ']'  → closing → pop '[', matches ']'  ✓   stack: ['(', '{']
Step 5: '}'  → closing → pop '{', matches '}'  ✓   stack: ['(']
Step 6: ')'  → closing → pop '(', matches ')'  ✓   stack: []

Stack is empty after all chars → VALID ✓

Step-by-Step Trace on "({])"

Input: ( { ] )

Step 1: '('  → push        stack: ['(']
Step 2: '{'  → push        stack: ['(', '{']
Step 3: ']'  → closing → pop '{', but ']' does NOT match '{'  ✗

Result: INVALID

import java.util.ArrayDeque;
import java.util.Deque;

public class BracketMatcher {

    static boolean isValid(String s) {
        Deque<Character> stack = new ArrayDeque<>();

        for (char c : s.toCharArray()) {
            // Push opening brackets
            if (c == '(' || c == '{' || c == '[') {
                stack.push(c);
            }
            // Check closing brackets
            else if (c == ')' || c == '}' || c == ']') {
                if (stack.isEmpty()) return false; // no matching opener

                char top = stack.pop();
                if (c == ')' && top != '(') return false;
                if (c == '}' && top != '{') return false;
                if (c == ']' && top != '[') return false;
            }
        }

        // Valid only if all openers were matched
        return stack.isEmpty();
    }

    public static void main(String[] args) {
        System.out.println(isValid("({[]})"));  // true
        System.out.println(isValid("({]}"));    // false
        System.out.println(isValid("((()"));    // false — unclosed
        System.out.println(isValid(""));        // true — empty string is valid
    }
}
🌍 Real World: Undo/Redo in every text editor (VS Code, Word, Photoshop) uses two stacks: an undo stack and a redo stack. Every action you take gets pushed to the undo stack. When you press Ctrl+Z, the action is popped from undo and pushed to redo. You use this data structure dozens of times daily.
ℹ️ The bracket-matching problem is O(n) time and O(n) space (worst case all openers). It appears frequently in coding interviews and is a classic demonstration of why stacks exist.
⚠️ Do not use the legacy java.util.Stack class. It extends Vector, which is synchronized on every operation — adding overhead you do not need in single-threaded code. Use ArrayDeque instead.
