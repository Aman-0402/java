Chapter 22: Recursion

Recursion is the technique of writing a function that solves a problem by calling itself on a smaller version of the same problem.

📜 History: The concept of recursion in mathematics predates computers — it appears in recursive definitions of factorial going back to the 1600s. In computing, it was first formalized in the 1950s with LISP (1958), the second-oldest programming language still in use.

What Is Recursion?

Imagine you are standing in a line and someone asks "what position are you in?" You do not want to count everyone in front, so you ask the person ahead: "What is your position?" They ask the person ahead of them, and so on — until someone at the front says "I am number 1." Then each person adds 1 and passes the answer back. That chain of asking-and-answering is recursion.

In code, a recursive method calls itself with a simpler input, until it reaches a stopping point called the base case.

The Two Rules of Recursion

  1. Base case — a condition where the function returns immediately without calling itself. Without this, you get infinite recursion and a crash.
  2. Recursive case — the function calls itself with input that is closer to the base case than the current input.
⚠️ Forgetting the base case is the #1 recursion mistake. Always write the base case first, before the recursive call, or your program will never stop.

The Call Stack — Visualised

Every time a method is called, Java pushes a stack frame onto the call stack. When the method returns, that frame is popped off. Recursive calls stack up frames and only unwind once the base case is hit.

factorial(4) is called:

 Call stack builds UP            Stack unwinds DOWN
 ┌───────────────────┐
 │  factorial(1) = 1 │  ← base case returns 1
 ├───────────────────┤
 │  factorial(2)     │  ← receives 1, returns 2 * 1 = 2
 ├───────────────────┤
 │  factorial(3)     │  ← receives 2, returns 3 * 2 = 6
 ├───────────────────┤
 │  factorial(4)     │  ← receives 6, returns 4 * 6 = 24
 └───────────────────┘  ← final answer: 24

Example 1: Factorial

The factorial of n (written n!) is n × (n-1) × (n-2) × … × 1. For example, 4! = 4 × 3 × 2 × 1 = 24. Notice that 4! = 4 × 3! — the problem contains a smaller copy of itself.

int factorial(int n) {
    // Base case — factorial(0) = 1 by definition
    if (n == 0) return 1;

    // Recursive case — n! = n * (n-1)!
    return n * factorial(n - 1);
}

System.out.println(factorial(5)); // 120

Step-by-Step Trace for factorial(4)

factorial(4)
  └─ 4 * factorial(3)
           └─ 3 * factorial(2)
                    └─ 2 * factorial(1)
                              └─ 1 * factorial(0)
                                        └─ returns 1   (base case)

Unwinding:
  factorial(1) = 1 * 1 = 1
  factorial(2) = 2 * 1 = 2
  factorial(3) = 3 * 2 = 6
  factorial(4) = 4 * 6 = 24

Example 2: Fibonacci

The Fibonacci sequence is 0, 1, 1, 2, 3, 5, 8, 13 … Each number is the sum of the two before it. fib(n) = fib(n-1) + fib(n-2). This has two recursive calls.

int fibonacci(int n) {
    // Base cases
    if (n == 0) return 0;
    if (n == 1) return 1;

    // Recursive case
    return fibonacci(n - 1) + fibonacci(n - 2);
}

System.out.println(fibonacci(6)); // 8

Step-by-Step Trace for fibonacci(4)

fib(4)
  ├─ fib(3)
  │    ├─ fib(2)
  │    │    ├─ fib(1) = 1
  │    │    └─ fib(0) = 0   → fib(2) = 1
  │    └─ fib(1) = 1        → fib(3) = 2
  └─ fib(2)
       ├─ fib(1) = 1
       └─ fib(0) = 0        → fib(2) = 1

fib(4) = 2 + 1 = 3
⚠️ The naive recursive Fibonacci is O(2ⁿ) — it recalculates the same values repeatedly. For fib(50) it makes over a trillion calls. Use a loop or memoisation for real use.
🔍 Fun Fact: Every recursive algorithm can be converted to an iterative one using a stack (and vice versa). The JVM actually converts some tail-recursive calls to loops internally. Functional languages like Haskell use recursion for almost everything — no loops at all.

Danger: Stack Overflow

Java's call stack has a limited size. If recursion goes too deep (typically beyond ~10,000 levels), you get a StackOverflowError. This is almost always caused by a missing or unreachable base case.

// BROKEN — goes -1, -2, -3 ... forever for negative input
int brokenFactorial(int n) {
    if (n == 0) return 1;
    return n * brokenFactorial(n - 1);
}

// FIXED — guard handles 0 and negative
int safeFactorial(int n) {
    if (n <= 0) return 1;
    return n * safeFactorial(n - 1);
}

Recursion vs Loops

Anything you can do with recursion you can also do with a loop, and vice versa. The choice is about clarity.

SituationPreferReason
Simple iteration (sum, max)LoopLess overhead, easier to read
Tree / graph traversalRecursionTrees are naturally recursive structures
Divide and conquer (merge sort)RecursionAlgorithm is naturally recursive
Very deep input (n > 10,000)LoopAvoid StackOverflowError

Full Working Example

public class RecursionDemo {

    static void countdown(int n) {
        if (n <= 0) {               // base case
            System.out.println("Blastoff!");
            return;
        }
        System.out.println(n);
        countdown(n - 1);           // recursive case
    }

    static int factorial(int n) {
        if (n <= 0) return 1;
        return n * factorial(n - 1);
    }

    static int fibonacci(int n) {
        if (n == 0) return 0;
        if (n == 1) return 1;
        return fibonacci(n - 1) + fibonacci(n - 2);
    }

    public static void main(String[] args) {
        countdown(5);
        // Prints: 5, 4, 3, 2, 1, Blastoff!

        System.out.println("5! = " + factorial(5));     // 120
        System.out.println("fib(8) = " + fibonacci(8)); // 21
    }
}
🌍 Real World: Git uses recursion to traverse your project's directory tree. When you run git add ., Git recursively walks every folder and subfolder to find changed files. File system traversal — one of the most common operations in software — is inherently recursive.
💡 When writing a recursive method, always start by asking: "What is the simplest input this function can receive?" That is your base case. Then ask: "How does the big problem break into a smaller copy of itself?" That is your recursive case.

🧠 Multiple Choice Questions

1. What are the two essential parts of a recursive method?

  • A) A loop and a condition
  • B) A base case and a recursive case
  • C) An input and an output
  • D) A stack and a queue
Show Answer

✅ B) A base case and a recursive case — Without a base case, recursion is infinite. Without a recursive case, it doesn't recurse.

2. What happens when recursion has no base case?

  • A) The method returns 0
  • B) The program compiles but prints nothing
  • C) StackOverflowError is thrown
  • D) The JVM restarts
Show Answer

✅ C) StackOverflowError is thrown — Each call adds a stack frame. Without a base case, the stack fills up and Java throws StackOverflowError.

3. What is the time complexity of computing fibonacci(n) using naive recursion?

  • A) O(n)
  • B) O(log n)
  • C) O(2^n)
  • D) O(n²)
Show Answer

✅ C) O(2^n) — Each call branches into two more calls, creating an exponential tree of redundant computations.

4. What is memoization?

  • A) A type of memory leak
  • B) Caching the results of expensive function calls to avoid redundant computation
  • C) A sorting technique
  • D) A way to reduce stack size
Show Answer

✅ B) Caching the results of expensive function calls to avoid redundant computation — Memoizing fibonacci reduces O(2^n) to O(n) by storing previously computed values.
