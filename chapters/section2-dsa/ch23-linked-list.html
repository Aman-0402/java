Chapter 23: Linked List

A linked list is a chain of nodes where each node holds a value and a pointer to the next node, making insertions and deletions fast without shifting elements.

The Train Carriages Analogy

Think of a linked list as a train. Each carriage (node) carries cargo (data) and is hooked to the next carriage. To find carriage 5, you must walk through carriages 1, 2, 3, 4 first — there is no shortcut. But adding or removing a carriage in the middle only requires unhooking a couple of connections, no matter how long the train is.

This is very different from an array, which is like a row of numbered seats: you can jump to seat 5 instantly, but adding a seat in the middle means physically moving every seat after it.

The Node Class

Every element in a linked list is a Node. A node has two fields: the data it holds, and a reference (next) pointing to the next node. The last node's next is null.

class Node {
    int data;   // the value stored in this node
    Node next;  // pointer to the next node (null if last)

    Node(int data) {
        this.data = data;
        this.next = null;
    }
}

Structure of a Singly Linked List

head
 │
 ▼
┌──────┬──────┐    ┌──────┬──────┐    ┌──────┬──────┐
│  10  │  ──────►  │  20  │  ──────►  │  30  │ null │
└──────┴──────┘    └──────┴──────┘    └──────┴──────┘
  Node 1               Node 2               Node 3

Each box: [data | next pointer]
The last node's next pointer is null (end of list).

Building a Linked List from Scratch

The list itself only needs to track the head — the first node. All other nodes are reachable by following next pointers.

public class LinkedList {
    Node head; // first node; null when list is empty

    // Add a node at the end of the list
    void add(int data) {
        Node newNode = new Node(data);
        if (head == null) {
            head = newNode;    // list was empty
            return;
        }
        // Walk to the last node
        Node current = head;
        while (current.next != null) {
            current = current.next;
        }
        current.next = newNode; // attach new node at the end
    }

    // Print all values
    void print() {
        Node current = head;
        while (current != null) {
            System.out.print(current.data + " -> ");
            current = current.next;
        }
        System.out.println("null");
    }

    // Count nodes
    int size() {
        int count = 0;
        Node current = head;
        while (current != null) {
            count++;
            current = current.next;
        }
        return count;
    }

    // Delete first node with matching value
    void delete(int data) {
        if (head == null) return;

        // Special case: delete head
        if (head.data == data) {
            head = head.next;
            return;
        }

        // Find the node just before the one to delete
        Node current = head;
        while (current.next != null) {
            if (current.next.data == data) {
                current.next = current.next.next; // skip over deleted node
                return;
            }
            current = current.next;
        }
    }
}

Step-by-Step: delete(20) on the list [10 → 20 → 30]

Before: head → [10|●] → [20|●] → [30|null]

1. head.data (10) != 20, so head stays.
2. Walk: current = node(10)
3. current.next.data = 20 — match found!
4. current.next = current.next.next
   i.e. node(10).next = node(30)

After:  head → [10|●] → [30|null]
   node(20) is now unreachable — garbage collected.

Full Working Example

public class LinkedListDemo {
    public static void main(String[] args) {
        LinkedList list = new LinkedList();
        list.add(10);
        list.add(20);
        list.add(30);
        list.add(40);

        list.print();          // 10 -> 20 -> 30 -> 40 -> null
        System.out.println("Size: " + list.size()); // 4

        list.delete(20);
        list.print();          // 10 -> 30 -> 40 -> null

        list.delete(10);       // delete head
        list.print();          // 30 -> 40 -> null
    }
}

Array vs Linked List

Operation Array Linked List
Access by indexO(1) — instantO(n) — must walk from head
Insert at endO(1) amortisedO(n) — must walk to end
Insert at beginningO(n) — shift allO(1) — update head
Delete in middleO(n) — shift allO(n) — walk, then O(1) unlink
MemoryCompact, contiguousExtra memory per node for pointer
💡 Use a linked list when you need frequent insertions or deletions at the beginning or middle of a collection. Use an array (or ArrayList) when you need fast random access by index.

Java's Built-in LinkedList

Java provides java.util.LinkedList in the Collections framework. It is a doubly linked list (each node also has a prev pointer) and implements both List and Deque.

import java.util.LinkedList;

LinkedList<Integer> list = new LinkedList<>();
list.add(10);         // add to end
list.addFirst(5);     // add to beginning
list.removeLast();    // remove from end
System.out.println(list.get(0)); // access by index (O(n) internally)
ℹ️ In practice, ArrayList outperforms LinkedList for most tasks because modern CPUs cache contiguous memory very efficiently. Prefer ArrayList unless you specifically need O(1) front insertions or are implementing a deque/queue.
⚠️ Never modify a linked list while iterating over it with a standard loop. Use an Iterator and its remove() method, or collect indices to delete afterward.
